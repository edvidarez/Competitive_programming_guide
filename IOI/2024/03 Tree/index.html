Tarea 3 — Tree

PDF original (oficial): statement.pdf

El problema

Tenemos un árbol con \(N\) vértices, numerados de \(0\) a \(N-1\). El vértice \(0\) es la raíz. Todo vértice, salvo la raíz, tiene un único padre. Para cada \(i\) con \(1 \le i \lt N\), el padre del vértice \(i\) es el vértice \(P[i]\), donde \(P[i] \lt i\). Además se asume \(P[0] = -1\).

Para cualquier vértice \(i\) (con \(0 \le i \lt N\)), el subárbol de \(i\) es el conjunto formado por:

  • el propio \(i\), y
  • cualquier vértice cuyo padre sea \(i\), y
  • cualquier vértice cuyo padre del padre sea \(i\), y
  • cualquier vértice cuyo padre del padre del padre sea \(i\), y
  • etcétera.

La figura de abajo muestra un árbol de ejemplo con \(N = 6\) vértices. Cada flecha conecta un vértice con su padre, salvo la raíz, que no tiene padre. El subárbol del vértice \(2\) contiene los vértices \(2, 3, 4\) y \(5\). El subárbol del vértice \(0\) contiene los \(6\) vértices del árbol, y el subárbol del vértice \(4\) contiene únicamente al vértice \(4\).

A cada vértice se le asigna un peso entero no negativo. Denotamos el peso del vértice \(i\) (con \(0 \le i \lt N\)) por \(W[i]\).

Tu tarea es escribir un programa que responda \(Q\) consultas, cada una dada por un par de enteros positivos \((L, R)\). La respuesta a una consulta se calcula así.

Considera asignar un entero, llamado coeficiente, a cada vértice del árbol. Tal asignación se describe con una sucesión \(C[0], \dots, C[N-1]\), donde \(C[i]\) (con \(0 \le i \lt N\)) es el coeficiente asignado al vértice \(i\). A esta sucesión la llamamos sucesión de coeficientes. Nota que los elementos de la sucesión pueden ser negativos, cero o positivos.

Para una consulta \((L, R)\), una sucesión de coeficientes se llama válida si, para todo vértice \(i\) (con \(0 \le i \lt N\)), se cumple lo siguiente: la suma de los coeficientes de los vértices que están en el subárbol de \(i\) no es menor que \(L\) ni mayor que \(R\). Es decir, si \(S[i]\) denota esa suma, entonces \(L \le S[i] \le R\) para todo \(i\).

Dada una sucesión de coeficientes \(C[0], \dots, C[N-1]\), el costo de un vértice \(i\) es \(|C[i]| \cdot W[i]\), donde \(|C[i]|\) es el valor absoluto de \(C[i]\). Finalmente, el costo total es la suma de los costos de todos los vértices. Tu tarea es calcular, para cada consulta, el costo total mínimo que se puede alcanzar con alguna sucesión de coeficientes válida.

Se puede demostrar que, para cualquier consulta, existe al menos una sucesión de coeficientes válida.

Detalles de implementación

Esta es una tarea funcional: no lees de la entrada estándar ni escribes a la salida estándar. Debes implementar los dos procedimientos siguientes, a los que el evaluador (grader) llama directamente.

Firmas a implementar 
void init(std::vector<int> P, std::vector<int> W);
long long query(int L, int R);
  • init(P, W): \(P\) y \(W\) son arreglos de enteros de longitud \(N\) que especifican los padres y los pesos. Este procedimiento se llama exactamente una vez al inicio de la interacción entre el grader y tu programa, en cada caso de prueba.
  • query(L, R): \(L\) y \(R\) son los enteros que describen una consulta. Este procedimiento se llama \(Q\) veces después de la invocación de init, en cada caso de prueba, y debe devolver la respuesta (un entero de 64 bits, long long) a la consulta dada.

Etiquetas Funcional Árboles Greedy Sumas de prefijos Convexidad

Resumen

Te dan un árbol enraizado con pesos \(W[i] \ge 0\). Para una consulta \((L, R)\) eliges coeficientes enteros \(C[i]\) (pueden ser negativos) tales que, para todo vértice, la suma \(S[i]\) de los coeficientes de su subárbol cumpla \(L \le S[i] \le R\). Implementa init y query: query(L,R) devuelve el costo mínimo \(\sum_i |C[i]| \cdot W[i]\). Hay hasta \(Q = 10^5\) consultas, así que cada una debe responderse muy rápido tras un preprocesamiento.

Lo que importa de los límites \(N \le 2 \cdot 10^5\), \(Q \le 10^5\): no puedes recalcular un greedy por consulta en el caso pleno; necesitas precómputo y luego query en \(O(1)\). Las subtareas con \(Q \le 10\) (1–3) sí permiten un greedy por consulta. Las subtareas por estructura del peso (\(W=1\), \(W \le 1\)) y por valor de \(L\) (\(L=1\)) revelan, una a una, la fórmula cerrada general.

Pistas

Pista 1 — ¿Cómo modelar la restricción?

¿Ya notaste que, en lugar de elegir los coeficientes \(C[i]\), te conviene elegir directamente las sumas de subárbol \(S[i]\)?

Define \(S[i] = \sum_{j \in \text{subárbol}(i)} C[j]\). Entonces, para un vértice \(i\) con hijos \(c_1, \dots, c_k\), se cumple \(\;C[i] = S[i] - \big(S[c_1] + \dots + S[c_k]\big)\). Es decir, fijar todas las \(S[i]\) determina todos los \(C[i]\). La restricción se vuelve simplemente \(L \le S[i] \le R\) para cada \(i\) (una variable independiente por vértice, dentro de una caja), y el costo es \[\sum_i \Big| S[i] - \textstyle\sum_{c \text{ hijo de } i} S[c] \Big| \cdot W[i].\] ¿Qué valor de \(S[\text{hoja}]\) es óptimo si una hoja no tiene hijos?

Pista 2 — Subtareas 1–3 (\(Q \le 10\)): un greedy de hojas a raíz

¿Y si recorres el árbol de las hojas hacia la raíz y solo bajas coeficientes cuando una suma se pasa de \(R\)?

Como \(L \ge 1\), cada hoja debe aportar al menos \(L\); ponerle exactamente \(S[\text{hoja}] = L\) es óptimo (cuesta \(L \cdot W[\text{hoja}]\) y no puede valer menos). Subiendo, la suma de un vértice interno arranca como la suma de sus hijos; si se pasa de \(R\), hay que restar. La estrategia (BottomUpGreedy): mientras \(S[v] \gt R\), decrementa en \(1\) el coeficiente del vértice de menor peso dentro del subárbol de \(v\) que todavía sea «válido» (que no rompa \(S \ge L\) en ningún ancestro). Es óptimo, pero hay que rehacerlo por consulta.

Pseudocódigo — BottomUpGreedy (por consulta) 
greedy(L, R):
  para cada v de las hojas hacia la raíz:
     si v es hoja:
        C[v] <- L ;  S[v] <- L
     si no:
        S[v] <- suma de S[hijos]          // con C[v] = 0 de momento
        mientras S[v] > R:
           elige el vértice k de MENOR peso en el subárbol de v
              tal que bajar su coeficiente NO viole S >= L en ningún ancestro
           C[k] <- C[k] - 1
           actualiza las sumas S afectadas (la de v y sus descendientes hasta k)
  regresa  suma de |C[v]| * W[v]

Subtarea 1 (\(W[P[i]] \le W[i]\), pesos no decrecientes hacia las hojas): el vértice válido de menor peso siempre se puede tomar como \(v\) mismo, así que el greedy es trivial. Subtarea 2 (\(N \le 2000\)): búsqueda explícita \(O(N^2)\) del mínimo válido. Subtarea 3 (\(N \le 60\,000\)): el mismo greedy pero hallando el mínimo válido con una estructura (un set, una cola de prioridad o un segment tree sobre el recorrido de Euler).

Pista 3 — Dos lemas que ordenan el problema

¿Notaste que siempre existe un óptimo donde las hojas valen \(L\) y los nodos internos solo «restan»?

  • Lema (hojas): existe un óptimo con \(C[\text{hoja}] = L\) (cada hoja paga \(L \cdot W[\text{hoja}]\)) y con \(C[\text{no-hoja}] \le 0\). El ajuste para no pasarse de \(R\) se hace bajando coeficientes solo en los nodos internos.
  • Lema (peso 0): si un nodo interno tiene peso \(0\), «cortarlo de sus hijos» no cambia la respuesta: los subárboles de sus hijos se vuelven instancias independientes cuyas respuestas se suman.

Estos dos hechos sugieren que la respuesta es una función «bonita» de \(L\) y \(R\), y que conviene procesar los vértices por peso. ¿Qué pasa si todos los pesos son iguales?

Pista 4 — Subtarea 4 (\(W = 1\)): una fórmula cerrada

Si todos los pesos valen \(1\) y \(k\) es el número de hojas, ¿cuánto cuesta forzar todas las restricciones?

Las \(k\) hojas aportan \(k \cdot L\) en total. La suma en la raíz vale \(k \cdot L\); si eso excede \(R\), hay que reabsorber \(k \cdot L - R\) unidades, y con peso \(1\) cada unidad cuesta \(1\). Por tanto: \[\text{respuesta} = k \cdot L + \max\big(0,\; k \cdot L - R\big).\] ¿Cómo extiendes esto cuando algunos pesos son \(0\) y otros \(1\)?

Pista 5 — Subtareas 5 y 6 (\(W \le 1\); \(L = 1\)) y el lema recursivo

¿Y si descompones el árbol cortando en los pesos \(0\) y luego restas el peso mínimo de los nodos internos?

Subtarea 5 (\(W \le 1\)): por el lema de peso \(0\), corta el árbol en cada nodo interno de peso \(0\); cada pedazo es una instancia con todos los pesos iguales a \(1\), así que aplicas la fórmula \(k \cdot L + \max(0, k\,L - R)\) en cada pedazo y sumas (con sumas de prefijos sobre el número de hojas).

Lema 5 (clave, general): sea \(k\) el número de hojas del subárbol y \(m\) el peso mínimo entre los nodos no hoja de ese subárbol. Si \(k \cdot L - R \gt 0\), el costo óptimo es \[(k \cdot L - R) \cdot m \; + \; s,\] donde \(s\) es el costo del mismo problema pero restándole \(m\) a todos los pesos no hoja. Al restar \(m\), el mínimo se vuelve \(0\): por el lema de peso \(0\) el árbol se parte y recursas en los pedazos.

Subtarea 6 (\(L = 1\)): es el caso particular de la fórmula general con \(L = 1\) (el cociente \(\lfloor R/L \rfloor\) se vuelve \(R\)); sirve para verificar que la maquinaria general funciona antes de soltar \(L\).

Pista final — Solución completa (100 puntos)

Idea

Aplica el Lema 5 repetidamente. En cada paso eliges el nodo no hoja de menor peso \(v\) del subárbol, emites un término \((k \cdot L - R) \cdot W'[v]\) (donde \(W'\) es el peso ya «rebajado»), restas \(W'[v]\) a todos los nodos no hoja —lo que vuelve \(0\) el mínimo— y, por el lema de peso \(0\), partes en los subárboles de los hijos y recursas. Lo esencial: qué nodo se elige y cómo se parte el árbol NO depende de \(L\) ni de \(R\), solo del orden de los pesos. Lo único que depende de la consulta es si cada término entra o no, según la condición \[k \cdot L - R \gt 0 \iff k \gt R/L.\]

Por eso basta una sola corrida genérica que produzca todos los términos \((k,\; \text{coeficiente } W')\). Agrupando por \(k \in [1, N]\) y usando sumas de prefijos, la respuesta a \((L,R)\) queda de la forma \(a_t \cdot L + b_t \cdot R\) con \(t = \min(N, \lfloor R/L \rfloor)\), más un término fijo \(L \cdot s\), donde \(s\) es la suma de los pesos de las hojas. Equivalentemente: la respuesta es \(L \cdot F(R/L)\), donde \(F\) es una función convexa, lineal a trozos, con quiebres en los enteros.

Implementación (Union-Find por peso descendente)

Ordena los vértices por peso descendente y usa Union-Find. Cada conjunto es un «clúster» activo con nleaves[r] = número de hojas y tcr[r] = peso de referencia actual. Al procesar un nodo no hoja \(v\), fusiona a sus hijos en \(v\) y acumula, para cada conteo de hojas \(k\), la pendiente sl[k] += (tcr - W[v]). Integrando esas pendientes obtienes ans[i] = el valor de \(F\) evaluado en el cociente entero \(i\) (con \(L\) factorizado). La respuesta total es \(L \cdot F(\lfloor R/L \rfloor)\); por la convexidad con quiebres enteros, query interpola linealmente entre ans[t] y ans[t+1] con \(t = \lfloor R/L \rfloor\).

Pseudocódigo — solución plena 
init(P, W):
  n <- longitud(P)
  construir lista de hijos e[] desde P
  orden <- vértices ordenados por W DESCENDENTE
  Union-Find:  par[v]=v ;  nleaves[v]=1 ;  tcr[v]=+infinito (n+1) ;  sl[0..n]=0
  v0 <- 0                      // costo base: suma de pesos de las hojas (por el factor L)
  para cada v en orden:
     si v es hoja (sin hijos):
        v0 <- v0 + W[v] ;  continuar
     r <- raíz(v)
     // al "activar" v como nodo interno de menor peso del clúster:
     sl[nleaves[r]] <- sl[nleaves[r]] + (tcr[r] - W[v])   // pendiente para ese conteo de hojas
     nleaves[r] <- nleaves[r] - 1
     tcr[r] <- W[v]                                       // nuevo peso de referencia (rebajado)
     para cada hijo u de v:
        sl[nleaves[u]] <- sl[nleaves[u]] + (tcr[u] - W[v])  // término del Lema 5 al fusionar
        unir u dentro de r ;  nleaves[r] <- nleaves[r] + nleaves[u]
  sl[nleaves[0]] <- sl[nleaves[0]] + tcr[0]               // cierre en la raíz
  // integrar pendientes para obtener F evaluada en cada cociente entero i:
  pendiente <- 0 ;  val <- v0
  para i desde n hacia 0:
     val  <- val + pendiente
     ans[i] <- val
     pendiente <- pendiente + sl[i]

query(L, R):
  t <- R / L                      // división entera = piso(R/L)
  si t >= n:  regresa  ans[n] * L            // zona plana: todo holgado
  resto <- R mod L
  resultado <- (L - resto) * ans[t]
  si resto != 0:
     resultado <- resultado + resto * ans[t+1]   // interpola: peso total L  =>  L * F(R/L)
  regresa resultado            // SIEMPRE int64

Por qué es correcto

La cadena de lemas justifica cada paso: (1) reducir al caso \(C[\text{hoja}] = L\), \(C[\text{no-hoja}] \le 0\); (2) el BottomUpGreedy es óptimo; (3) cortar en los pesos \(0\) descompone el problema en instancias independientes; (4) el cierre \(k \cdot L + \max(0, k\,L - R)\) cuando los pesos son uniformes; y (5) la descomposición recursiva \(\text{costo} = (k\,L - R) \cdot m + s\) restando el mínimo peso no hoja. La corrida genérica produce, sin mirar \(L\) ni \(R\), un multiconjunto de términos \((k, \text{coeficiente})\); un término entra exactamente cuando \(k \gt R/L\). Con sumas de prefijos eso da \(a_t \cdot L + b_t \cdot R\) con \(t = \min(N, \lfloor R/L \rfloor)\), y la implementación lo codifica como la integral de las pendientes sl[] sobre el conteo de hojas, evaluada con interpolación lineal por la convexidad. Verificado contra las 71 salidas oficiales del jurado (checker testlib): 71/71 OK.

Complejidad: init en \(O(N \log N)\) (domina el ordenar por peso; el Union-Find con compresión de caminos es casi lineal y la integración de pendientes es \(O(N)\)); query en \(O(1)\) (una división, un módulo y dos accesos a ans[] con interpolación). Total \(O(N \log N + Q)\), memoria \(O(N)\). Con \(N \le 2 \cdot 10^5\) y \(Q \le 10^5\) corre en ~0.10 s frente al límite de 2.0 s.
