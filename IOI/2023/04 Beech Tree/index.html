Tarea 4 — Beech Tree (Árbol de haya)

PDF original (oficial): statement.pdf

El problema

El bosque de Vétyem es famoso por sus muchos árboles coloridos. Uno de los hayas más viejos y altos se llama Ős Vezér.

El árbol Ős Vezér se modela como un conjunto de \(N\) nodos y \(N-1\) aristas. Los nodos se numeran de \(0\) a \(N-1\) y las aristas de \(1\) a \(N-1\). Cada arista conecta dos nodos distintos. En concreto, la arista \(i\) (con \(1 \le i \lt N\)) conecta el nodo \(i\) con el nodo \(P[i]\), donde \(0 \le P[i] \lt i\). Al nodo \(P[i]\) se le llama el padre del nodo \(i\), y al nodo \(i\) se le llama un hijo de \(P[i]\).

Cada arista tiene un color. Hay \(M\) colores posibles, numerados de \(1\) a \(M\). El color de la arista \(i\) es \(C[i]\). Aristas distintas pueden tener el mismo color.

Nota que, según las definiciones anteriores, el caso \(i = 0\) no corresponde a una arista del árbol. Por comodidad, fijamos \(P[0] = -1\) y \(C[0] = 0\).

Por ejemplo, supón que Ős Vezér tiene \(N = 18\) nodos y \(M = 3\) colores posibles, con \(17\) aristas descritas por las conexiones \(P = [-1, 0, 0, 0, 1, 1, 1, 2, 2, 3, 3, 3, 4, 4, 5, 10, 11, 11]\) y los colores \(C = [0, 1, 2, 3, 1, 2, 3, 1, 3, 3, 2, 1, 1, 2, 2, 1, 2, 3]\). El árbol se ve así:

A Árpád, un guardabosques talentoso, le gusta estudiar partes específicas del árbol llamadas subárboles. Para cada \(r\) tal que \(0 \le r \lt N\), el subárbol del nodo \(r\) es el conjunto \(T(r)\) de nodos con las siguientes propiedades:

  • El nodo \(r\) pertenece a \(T(r)\).
  • Si un nodo \(x\) pertenece a \(T(r)\), entonces todos los hijos de \(x\) también pertenecen a \(T(r)\).
  • Ningún otro nodo pertenece a \(T(r)\).

El tamaño del conjunto \(T(r)\) se denota \(\lvert T(r) \rvert\).

Árpád descubrió hace poco una propiedad de los subárboles complicada pero interesante. Su descubrimiento requirió mucho jugar con papel y lápiz, y sospecha que tú también necesitarás hacerlo para entenderla.

Supón que tenemos un \(r\) fijo y una permutación \(v_0, v_1, \ldots, v_{\lvert T(r) \rvert - 1}\) de los nodos del subárbol \(T(r)\).

Para cada \(i\) tal que \(1 \le i \lt \lvert T(r) \rvert\), sea \(f(i)\) el número de veces que el color \(C[v_i]\) aparece en la siguiente secuencia de \(i-1\) colores: \(C[v_1], C[v_2], \ldots, C[v_{i-1}]\).

(Observa que \(f(1)\) siempre vale \(0\), porque la secuencia de su definición es vacía.)

La permutación \(v_0, v_1, \ldots, v_{\lvert T(r) \rvert - 1}\) es una permutación hermosa si y solo si se cumplen todas estas propiedades:

  • \(v_0 = r\).
  • Para cada \(i\) tal que \(1 \le i \lt \lvert T(r) \rvert\), el padre del nodo \(v_i\) es el nodo \(v_{f(i)}\).

Para cualquier \(r\) con \(0 \le r \lt N\), el subárbol \(T(r)\) es un subárbol hermoso si y solo si existe una permutación hermosa de los nodos de \(T(r)\). Nota que, por la definición, todo subárbol formado por un solo nodo es hermoso.

En el árbol de ejemplo de arriba se puede mostrar que los subárboles \(T(0)\) y \(T(3)\) no son hermosos. El subárbol \(T(14)\) sí es hermoso, pues consta de un solo nodo. A continuación mostramos que el subárbol \(T(1)\) también es hermoso.

Considera la secuencia de enteros distintos \([v_0, v_1, v_2, v_3, v_4, v_5, v_6] = [1, 4, 5, 12, 13, 6, 14]\). Esta secuencia es una permutación de los nodos de \(T(1)\). La siguiente figura la muestra: las etiquetas pegadas a los nodos son los índices en los que aparecen en la permutación.

Verifiquemos que en efecto es una permutación hermosa:

  • \(v_0 = 1\).
  • \(f(1) = 0\) porque \(C[v_1] = C[4] = 1\) aparece \(0\) veces en la secuencia \([\,]\). El padre de \(v_1\) es \(v_0\): el padre del nodo \(4\) es el \(1\) (\(P[4]=1\)).
  • \(f(2) = 0\) porque \(C[v_2] = C[5] = 2\) aparece \(0\) veces en \([1]\). El padre de \(v_2\) es \(v_0\): el padre del \(5\) es el \(1\).
  • \(f(3) = 1\) porque \(C[v_3] = C[12] = 1\) aparece \(1\) vez en \([1, 2]\). El padre de \(v_3\) es \(v_1\): el padre del \(12\) es el \(4\).
  • \(f(4) = 1\) porque \(C[v_4] = C[13] = 2\) aparece \(1\) vez en \([1, 2, 1]\). El padre de \(v_4\) es \(v_1\): el padre del \(13\) es el \(4\).
  • \(f(5) = 0\) porque \(C[v_5] = C[6] = 3\) aparece \(0\) veces en \([1, 2, 1, 2]\). El padre de \(v_5\) es \(v_0\): el padre del \(6\) es el \(1\).
  • \(f(6) = 2\) porque \(C[v_6] = C[14] = 2\) aparece \(2\) veces en \([1, 2, 1, 2, 3]\). El padre de \(v_6\) es \(v_2\): el padre del \(14\) es el \(5\).

Como pudimos encontrar una permutación hermosa de los nodos de \(T(1)\), el subárbol \(T(1)\) es hermoso.

Tu tarea es ayudar a Árpád a decidir, para cada subárbol de Ős Vezér, si es hermoso o no.

Detalles de implementación

Debes implementar el siguiente procedimiento. El evaluador lo llama exactamente una vez por cada caso de prueba:

Firma a implementar 
int[] beechtree(int N, int M, int[] P, int[] C)
  • \(N\): el número de nodos del árbol.
  • \(M\): el número de colores posibles para las aristas.
  • \(P\), \(C\): arreglos de longitud \(N\) que describen las aristas (con \(P[0]=-1\) y \(C[0]=0\)).
  • El procedimiento debe regresar un arreglo \(b\) de longitud \(N\). Para cada \(r\) con \(0 \le r \lt N\), \(b[r]\) debe ser \(1\) si \(T(r)\) es hermoso, y \(0\) en caso contrario.

Resumen

Te dan un árbol enraizado en \(0\) con \(N\) nodos; la arista de cada nodo \(i\) a su padre \(P[i]\) tiene un color \(C[i]\). Para cada nodo \(r\) debes decidir si su subárbol \(T(r)\) admite una permutación hermosa: un orden \(v_0=r, v_1, \ldots\) donde el padre de \(v_i\) es \(v_{f(i)}\), y \(f(i)\) cuenta cuántas veces ya apareció el color \(C[v_i]\) antes. Devuelve un arreglo \(0/1\) de longitud \(N\).

Lo que importa de los límites \(N, M \le 2\cdot10^5\) exige una solución casi lineal: la final es \(O(N\log^2 N)\). La subtarea 1 (\(N \le 8\)) admite fuerza bruta probando todas las permutaciones. Las subtareas de línea, profundidad \(\le 2\), colorTwice y \(M=2\) están pensadas para que descubras primero la condición necesaria y luego la caracterización completa.

Pistas

Pista 1 — Entender la regla \(f(i)\)

¿Ya tradujiste qué significa exactamente que el padre de \(v_i\) sea \(v_{f(i)}\)? ¿Qué controla la posición \(f(i)\)?

Cuando insertas \(v_i\), miras su color \(c = C[v_i]\) (el color de la arista que va de \(v_i\) a su padre real). \(f(i)\) es cuántas veces ya apareció ese color \(c\) entre \(v_1, \ldots, v_{i-1}\). La regla pide que el padre real de \(v_i\) sea justo el nodo que ocupa la posición \(f(i)\) de la permutación. Piensa en el color \(c\): su primera aparición debe "colgar" de \(v_0=r\) (porque ahí \(f=0\)); la \(k\)-ésima aparición de \(c\) debe colgar del nodo donde quedó la \((k-1)\)-ésima. Es decir, los nodos cuya arista al padre es de color \(c\) se van encadenando.

Pista 2 — Subtarea 1 (\(N \le 8\)): fuerza bruta

Con \(\lvert T(r) \rvert \le 8\), ¿cuántas permutaciones hay? ¿Puedes probarlas todas y verificar la regla directamente?

Para cada \(r\), genera todas las permutaciones de \(T(r)\) que empiezan en \(r\), calcula \(f(i)\) con un contador de colores y revisa si el padre de cada \(v_i\) es \(v_{f(i)}\). Como \(\lvert T(r) \rvert \le 8\), son a lo más \(7! = 5040\) permutaciones por subárbol: trivial. Esto te da los \(9\) puntos de la subtarea 1, pero no escala (es exponencial). Úsalo solo para validar ideas en chiquito.

Pista 3 — La condición NECESARIA: hermanos con colores distintos

¿Qué pasa si un nodo \(x\) tiene dos hijos cuyas aristas son del mismo color \(c\)? Cuando insertas el segundo, ¿a quién "apunta" \(f\)?

Supón que \(x\) tiene dos hijos con aristas de color \(c\). Al insertar el primero, \(f\) cuenta \(0\) apariciones previas de \(c\)… pero al insertar el segundo, el color \(c\) ya apareció al menos una vez, así que \(f \ge 1\) y la regla obligaría a que su padre sea el nodo de la primera aparición de \(c\), que no es \(x\). Contradicción. Por lo tanto:

Condición necesaria: si algún nodo de \(T(r)\) tiene dos hijos con aristas del mismo color, entonces \(T(r)\) no es hermoso.

Esta sola condición ya resuelve varias subtareas fáciles (línea, profundidad \(\le 2\), muchos casos de \(M=2\) y de colorTwice): recórrela hacia arriba y marca como no-hermoso cualquier subárbol que contenga un nodo con hermanos del mismo color. ¿Pero es suficiente?

Pista 4 — ¿La condición necesaria basta? El "anidamiento" por tamaño

Aunque todos los hermanos tengan colores distintos, ¿siempre existe un orden de inserción válido? ¿Qué relación deben guardar entre sí los subárboles de dos nodos para poder intercalarse?

No siempre basta. Recuerda la Pista 1: para cada color \(c\), los nodos con arista de color \(c\) se encadenan; la \(k\)-ésima cuelga de donde quedó la \((k-1)\)-ésima. Eso impone un orden de inserción coherente para todo el subárbol a la vez. La forma cómoda de capturarlo: ordena los nodos de \(T(u)\) por el tamaño de su propio subárbol. Para que un nodo "pequeño" \(x\) pueda insertarse antes (o anidarse dentro) de uno "grande" \(y\) (con \(\mathrm{sz}[x] \le \mathrm{sz}[y]\)), hace falta que cada color presente entre los hijos de \(x\) también esté entre los hijos de \(y\), y que el hijo de \(x\) con ese color tenga subárbol no más grande que el hijo correspondiente de \(y\). Llama a esa relación \(\mathrm{comp}(x, y)\).

La idea final: al unir todos los descendientes de \(u\) manteniéndolos ordenados por tamaño, cada par consecutivo debe cumplir \(\mathrm{comp}\). Si algo lo viola, \(T(u)\) no es hermoso.

Pista final — Solución completa (100 puntos)

Idea. Procesamos los nodos de abajo hacia arriba. Para un nodo \(u\), \(T(u)\) es hermoso si y solo si se cumplen las tres cosas:

  1. (a) Hermanos distintos: todos los hijos directos de \(u\) tienen colores de arista distintos (la condición necesaria de la Pista 3, propagada hacia arriba).
  2. (b) Hijos hermosos: el subárbol de cada hijo de \(u\) es a su vez hermoso.
  3. (c) Anidamiento consistente: al fusionar todos los nodos de los subárboles de los hijos de \(u\) en un solo conjunto ordenado por tamaño de subárbol, cada par consecutivo \((\text{pred}, \text{suc})\) cumple \(\mathrm{comp}(\text{pred}, \text{suc})\), donde

\(\mathrm{comp}(x, y)\) (con \(\mathrm{sz}[x] \le \mathrm{sz}[y]\)) es verdadero cuando todo color presente entre las aristas-hijo de \(x\) también está entre las de \(y\), y para ese color el tamaño del subárbol del hijo de \(x\) es \(\le\) el del hijo correspondiente de \(y\).

Por qué es correcto. La necesidad de (a) está demostrada en la Pista 3 (dos hermanos del mismo color fuerzan \(f \ge 1\) apuntando a un nodo equivocado). La suficiencia (la caracterización completa, condición (c)) corresponde al editorial oficial: la relación de "anidamiento por tamaño" sobre el orden por tamaño de subárbol captura exactamente cuándo existe un orden de inserción válido que respeta \(f(i)\) para todos los colores a la vez. La implementación de referencia reproduce los tres ejemplos del enunciado (\([0,1,1,1]\); el arreglo de \(18\) con \(T(0),T(3)\) no hermosos; y \([0,1,1,1,1,1,1]\)) y pasa los \(158\) casos oficiales que cubren las \(9\) subtareas, incluyendo \(N = 2\cdot10^5\).

Cómo implementarlo eficiente. Para cada \(u\) mantén un conjunto ordenado \(\mathrm{ord}[u]\) con los pares \((\mathrm{sz}, \text{nodo})\) de su subárbol, ordenado por tamaño. Al subir de un hijo \(v\) a su padre \(u\), fusiona \(\mathrm{ord}[v]\) dentro de \(\mathrm{ord}[u]\) con la técnica small-to-large (DSU on tree): siempre insertas el conjunto más chico en el más grande. Cada vez que insertas un elemento, basta verificar \(\mathrm{comp}\) contra su vecino anterior y su vecino siguiente en el orden por tamaño; si alguna falla, marca \(t[u]=0\). Cada nodo se mueve \(O(\log N)\) veces (small-to-large), y cada verificación de \(\mathrm{comp}\) recorre el mapa de colores de los hijos en otro factor logarítmico amortizado.

Pseudocódigo 
funcion beechtree(N, M, P, C):
    t[0..N-1] = 1                 # 1 = hermoso (por defecto)
    sz[u] = 1 para todo u
    hijos[u] = lista vacia
    colorHijo[u] = mapa vacio     # color -> nodo hijo de u con ese color

    para v de N-1 hasta 1:        # los padres tienen indice menor que sus hijos
        u = P[v]
        sz[u] = sz[u] + sz[v]
        agrega v a hijos[u]
        si C[v] ya esta en colorHijo[u]:
            t[u] = 0              # dos hijos de u con el mismo color (condicion necesaria)
        colorHijo[u][C[v]] = v

    DFS(0)
    regresa t

funcion comp(x, y):              # supone sz[x] <= sz[y]; devuelve si x "se anida" en y
    para cada (color c, hijo sx) en colorHijo[x]:
        si c no esta en colorHijo[y]:
            regresa falso
        sy = colorHijo[y][c]
        si sz[sx] > sz[sy]:
            regresa falso
    regresa verdadero

funcion fusiona(u, v):           # mete ord[v] dentro de ord[u] (small-to-large)
    si tamano(ord[u]) < tamano(ord[v]):
        intercambia ord[u] y ord[v]
    para cada (s, x) en ord[v]:           # recorrido en orden por (tamano, nodo)
        inserta (s, x) en ord[u]          # conjunto ordenado por tamano
        pred = vecino anterior de x en ord[u]   (si existe)
        suc  = vecino siguiente de x en ord[u]  (si existe)
        si existe pred y NO comp(pred, x):
            regresa falso
        si existe suc  y NO comp(x, suc):
            regresa falso
    regresa verdadero

funcion DFS(u):
    ord[u] = { (sz[u], u) }       # conjunto ordenado por (tamano de subarbol, nodo)
    para cada hijo v en hijos[u]:
        DFS(v)
        si t[v] == 0 O NO fusiona(u, v):
            t[u] = 0

# Subtarea 1 (N <= 8): alternativa de fuerza bruta -> probar todas las
# permutaciones de cada T(r) que empiezan en r y verificar la regla del
# padre v_{f(i)}. NO usar para N grande: es exponencial.

Complejidad: tiempo \(O(N\log^2 N)\) y memoria \(O(N)\) (más las estructuras de conjuntos/mapas, \(O(N)\)). El factor \(\log^2\) sale del small-to-large (un \(\log\)) combinado con el costo de \(\mathrm{comp}\) (otro \(\log\) amortizado). Para \(N = 2\cdot10^5\) sobra: medido \(\lesssim 0.30\) s (máximo entre los \(158\) casos) frente al límite de \(1.5\) s. La condición necesaria sola es \(O(N)\).
