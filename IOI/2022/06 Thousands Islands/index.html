Tarea 6 — Mil Islas (Thousands Islands)

PDF original (oficial): statement.pdf

El problema

Las Mil Islas son un grupo de hermosas islas en el Mar de Java. Hay \(N\) islas, numeradas de \(0\) a \(N-1\).

Existen \(M\) canoas, numeradas de \(0\) a \(M-1\), que sirven para navegar entre islas. Para cada \(i\) con \(0 \le i \le M-1\), la canoa \(i\) puede estar atracada en la isla \(U[i]\) o en la isla \(V[i]\), y sirve para navegar entre \(U[i]\) y \(V[i]\). En concreto:

  • Cuando la canoa está atracada en \(U[i]\), se puede usar para navegar de \(U[i]\) a \(V[i]\); tras hacerlo, la canoa queda atracada en \(V[i]\).
  • Cuando la canoa está atracada en \(V[i]\), se puede usar para navegar de \(V[i]\) a \(U[i]\); tras hacerlo, la canoa queda atracada en \(U[i]\).

Inicialmente cada canoa \(i\) está atracada en la isla \(U[i]\). Puede haber varias canoas que conecten el mismo par de islas, y varias canoas atracadas en la misma isla.

Por seguridad, una canoa necesita mantenimiento después de cada uso, lo que prohíbe usar la misma canoa dos veces seguidas: tras usar una canoa \(i\), hay que usar otra canoa antes de poder volver a usar la \(i\).

Bu Dengklek quiere planear un viaje por algunas de las islas. Su viaje es válido si y solo si se cumplen todas estas condiciones:

  • Empieza y termina su viaje en la isla \(0\).
  • Visita al menos una isla distinta de la isla \(0\).
  • Al terminar el viaje, cada canoa queda atracada en la misma isla en la que estaba antes de empezar. Es decir, la canoa \(i\), para cada \(0 \le i \le M-1\), debe quedar atracada en la isla \(U[i]\).

Ayuda a Bu Dengklek a encontrar cualquier viaje válido que use a lo sumo \(2\,000\,000\) navegaciones, o a determinar que no existe ninguno. Se puede demostrar que, bajo las restricciones de esta tarea (ver la sección de Límites), si existe un viaje válido, también existe uno que no usa más de \(2\,000\,000\) navegaciones.

Detalles de implementación

No lees de la entrada estándar ni escribes en la salida estándar: debes implementar una función que el evaluador llamará exactamente una vez.

Firma a implementar 
variant<bool, int[]> find_journey(int N, int M, int[] U, int[] V)
  • \(N\): el número de islas.
  • \(M\): el número de canoas.
  • \(U\), \(V\): arreglos de longitud \(M\) que describen las canoas.
  • La función debe devolver un booleano o un arreglo de enteros:
    • Si no existe viaje válido, debe devolver false.
    • Si existe viaje válido, tienes dos opciones:
      • Para el puntaje completo: devolver un arreglo de a lo sumo \(2\,000\,000\) enteros que representen un viaje válido. Los elementos del arreglo son los números de las canoas usadas, en el orden en que se usan.
      • Para un puntaje parcial: devolver true, un arreglo de más de \(2\,000\,000\) enteros, o un arreglo que no describa un viaje válido (ver la sección de Subtareas).
  • Esta función se llama exactamente una vez.

Ejemplo 1

Considera la siguiente llamada:

find_journey(4, 5, [0, 1, 2, 0, 3], [1, 2, 3, 3, 1])

Un viaje válido posible es el siguiente. Bu Dengklek navega primero las canoas \(0, 1, 2\) y \(4\) en ese orden; queda en la isla \(1\). Después puede volver a navegar la canoa \(0\), porque ahora está atracada en la isla \(1\) y la última canoa usada no fue la \(0\); tras usarla, llega a la isla \(0\). Sin embargo, las canoas \(1\), \(2\) y \(4\) no están atracadas donde estaban antes. Bu Dengklek continúa navegando las canoas \(3, 2, 1, 4\) y \(3\) de nuevo. Vuelve a la isla \(0\) y todas las canoas quedan atracadas como al principio.

Llamada

find_journey(4, 5,
   [0, 1, 2, 0, 3],
   [1, 2, 3, 3, 1])

Retorno

[0, 1, 2, 4, 0, 3, 2, 1, 4, 3]

Ejemplo 2

Considera la siguiente llamada:

find_journey(2, 3, [0, 1, 1], [1, 0, 0])

Bu Dengklek solo puede empezar navegando la canoa \(0\); después puede navegar la canoa \(1\) o la \(2\) (no puede usar la \(0\) dos veces seguidas). En ambos casos vuelve a la isla \(0\), pero las canoas no quedan atracadas donde estaban antes, y ya no puede navegar ninguna canoa: la única canoa atracada en la isla \(0\) es justo la que acaba de usar. Como no hay viaje válido, la función debe devolver false.

Llamada

find_journey(2, 3,
   [0, 1, 1],
   [1, 0, 0])

Retorno

false

Resumen

Cada canoa es una arista que, al usarse, se da la vuelta (cambia de atracadero). Hay que construir una caminata cerrada que empiece y termine en la isla \(0\), visite otra isla, no use la misma canoa dos veces seguidas y deje cada canoa atracada como al inicio; o demostrar que no existe y devolver false. Observación clave: dejar cada canoa restaurada equivale a usarla un número par de veces.

Lo que importa de los límites \(N \le 100\,000\) y \(M \le 200\,000\) piden algo casi lineal, del orden de \(O((N+M)\log M)\). Las subtareas dan estructura: \(N=2\) para entender la imposibilidad; grafo «completo», «bidireccional» y «doble» para descubrir la idea de recorrido + espejo antes del caso general.

Pistas

Pista 1 — ¿Cómo modelarlo?

¿Ya pensaste en esto como un grafo dirigido de atracaderos, donde cada canoa es una arista que cambia de sentido cada vez que la usas?

En cada momento, desde una isla solo puedes salir por las canoas atracadas ahí. Usar la canoa \(i\) atracada en \(U[i]\) te lleva \(U[i] \to V[i]\) y deja esa arista apuntando \(V[i] \to U[i]\). Mantén, por isla, la lista de canoas que ahora pueden salir de ella. ¿Qué propiedad debe cumplir la secuencia de canoas para que cada una quede restaurada al final?

Pista 2 — Subtarea 1 (\(N=2\)): ¿cuándo es imposible?

Con solo dos islas, ¿qué necesitas para salir de \(0\) y poder regresar dejando todo en su lugar? ¿Qué pasa si la isla \(0\) tiene una sola canoa de salida?

Si desde la isla \(0\) solo puede salir una canoa, la usas, cruzas, y para volver tendrías que reusarla de inmediato (prohibido) o quedarte atrapado. Mira el Ejemplo 2: la canoa \(0\) sale de \(0\), pero al regresar la única canoa en \(0\) es la que acabas de usar. La respuesta es false. La lección: una isla sin salida útil es un callejón sin salida.

Pista 3 — ¿Qué islas son inútiles? (poda)

¿Notaste que una isla sin ninguna canoa de salida no sirve para nada? Si llegas a ella no puedes seguir ni regresar. ¿Y si la borras, qué pasa con sus vecinas?

Borra en cadena toda isla con grado de salida \(0\) (como «pelar hojas»): al borrar una, sus predecesoras pueden quedarse también sin salida y se vuelven a encolar. Si después de podar la isla \(0\) se queda sin salida, no existe viaje y devuelves false. Esta poda nunca descarta soluciones, porque solo elimina islas que jamás podrían formar parte de un viaje válido.

Pista 4 — Subtareas 2–4: la idea de «recorrido + espejo»

En los grafos «completo», «bidireccional» y «doble» siempre tienes una segunda forma de cruzar entre dos islas. ¿Cómo aprovechas eso para que cada canoa termine usada un número par de veces?

Si haces un recorrido cerrado y luego lo repites en espejo (en orden inverso, deshaciendo cada movimiento), cada canoa se usa un número par de veces y queda restaurada. El único cuidado es no repetir la misma canoa dos veces seguidas en los empalmes: para eso necesitas dos recorridos distintos que salgan de la misma isla, de modo que puedas intercalarlos. ¿Qué isla te conviene como base?

Pista 5 — Tramos forzados vs. libertad

Si la isla en la que estás tiene exactamente una canoa de salida, ¿tienes opción? ¿Y si tiene dos o más?

Mientras la isla actual tenga grado de salida exactamente \(1\), estás obligado a tomar esa única canoa: la apuntas en un «prefijo de ida», borras la isla y avanzas. Más tarde regresarás por ese mismo prefijo invertido (cada canoa forzada se usa dos veces, así que queda restaurada). El verdadero trabajo —armar los dos recorridos espejados— empieza en la primera isla con grado de salida \(\ge 2\): ahí tienes la libertad necesaria para no repetir canoa.

Pista final — Solución completa (100 puntos)

Idea general. Modela cada canoa como una arista dirigida \(U[i] \to V[i]\) que se voltea al usarse; mantén por isla un conjunto de canoas que pueden salir de ella (y el inverso). La construcción tiene cuatro fases:

  1. Poda de callejones sin salida. Borra en cadena toda isla con grado de salida \(0\); al borrar una, encola sus predecesoras que se queden sin salida. Si tras podar la isla \(0\) no tiene salida, devuelve false.
  2. Prefijo forzado (ir y volver). Mientras la isla actual START tenga grado de salida exactamente \(1\), toma esa única canoa, regístrala en el prefijo de ida, borra la isla y avanza. El sufijo final será ese prefijo invertido: cada canoa forzada se usa \(2\) veces, así que queda restaurada y regresas por donde viniste.
  3. Núcleo en una isla \(A=\)START con grado de salida \(\ge 2\). Construye dos recorridos cerrados distintos que regresan a \(A\): el primero avanza tomando la primera salida en cada isla hasta cerrar un ciclo; el segundo arranca por otra salida distinta de \(A\) y se engancha de vuelta a la estructura del primero. A cada recorrido le agregas su espejo (recorrido inverso) para que toda canoa quede usada un número par de veces.
  4. Ensamble sin repetición. Concatena recorrido 1 + espejo 1 + canoa puente + recorrido 2 + espejo 2 + canoa puente. Donde un empalme dejaría la misma canoa dos veces seguidas, voltea o intercala (con una pila) el bloque conflictivo para romper la repetición.

El viaje final es: prefijo de ida + núcleo ensamblado + prefijo de ida invertido. Empieza y termina en \(0\), visita otra isla, restaura todas las canoas y nunca usa la misma canoa dos veces seguidas.

Por qué es correcto.

  • La poda solo elimina islas que nunca pueden formar parte de un viaje válido (no tienen salida), así que no descarta ninguna solución: si tras podar la isla \(0\) muere, no existe ruta restauradora y false es correcto (esto cubre \(N=2\) con una sola canoa en \(0\) y el Ejemplo 2).
  • Cada canoa del prefijo forzado y de los recorridos espejados se usa un número par de veces \(\Rightarrow\) cada canoa vuelve a su atracadero original y el viaje regresa a la isla \(0\).
  • La fase de ensamble (volteo/pila) garantiza que ninguna canoa se use dos veces seguidas.
  • El enunciado garantiza que si existe viaje, existe uno con \(\le 2\,000\,000\) pasos; esta construcción los produce muy por debajo del límite (en el peor caso oficial: \(299\,996\) navegaciones).
Pseudocódigo 
funcion find_journey(N, M, U, V):
    // construir el grafo dirigido de atracaderos
    para cada canoa i:
        agregar salida (i: U[i] -> V[i]) en out[U[i]]
        registrar i en in[V[i]]
    prefijo_ida <- []            // canoas del tramo forzado, en orden
    START <- 0

    repetir:
        // (1) PODA en cadena de callejones sin salida
        mientras exista isla con out vacio:
            borrar esa isla y todas sus aristas
            encolar las islas que queden con out vacio
        si out[START] esta vacio:
            devolver FALSE                  // no existe viaje

        // (2) TRAMO FORZADO: una sola salida -> estas obligado
        si |out[START]| == 1:
            c <- unica canoa de salida de START
            prefijo_ida <- prefijo_ida + [c]
            borrar la isla START
            START <- destino de c
            continuar el repetir

        // (3) NUCLEO: aqui |out[START]| >= 2
        A <- START
        recorrido_1 <- avanzar tomando la PRIMERA salida en cada isla
                       hasta cerrar un ciclo que vuelva a una isla ya vista
        recorrido_2 <- arrancar por OTRA salida distinta de A,
                       avanzar hasta engancharse con recorrido_1 y volver a A
        espejo_1 <- inverso(recorrido_1)
        espejo_2 <- inverso(recorrido_2)
        nucleo <- recorrido_1 + espejo_1 + puente
                + recorrido_2 + espejo_2 + puente
        si algun empalme repite canoa consecutiva:
            voltear / intercalar (con una pila) el bloque marcado

        // (4) ENSAMBLE: ir, nucleo, deshacer el prefijo
        viaje <- prefijo_ida + nucleo + invertir(prefijo_ida)
        devolver viaje

    // invariantes:
    //   - toda canoa se usa un numero PAR de veces (queda restaurada)
    //   - nunca dos usos consecutivos de la misma canoa
    //   - el viaje empieza y termina en la isla 0

Complejidad: \(O((N+M)\log M)\) en tiempo (conjuntos por isla con inserciones y borrados logarítmicos; cada arista se procesa un número constante de veces) y \(O(N+M)\) en memoria. La longitud del viaje generado es \(O(N+M)\) pasos, muy por debajo del límite de \(2\,000\,000\). Corre holgado dentro de \(1.0\) s y \(2\) GB.
