Problema 01 — Convertidor

Enunciado oficial: statement.md · Practica en línea: omegaUp

El problema

La numeración biyectiva es un sistema en el que todos los enteros positivos pueden ser representados de exactamente una forma.

La numeración biyectiva base 26 usa el alfabeto para representar 26 dígitos del 1 al 26 (A=1, B=2, C=3, ..., Z=26, AA=27, AB=28, ...). En la numeración biyectiva 26 no se usa la letra Ñ.

Esta numeración se usa en las hojas de cálculo.

Escribe un programa que convierta un entero del sistema decimal a una cadena que representa su valor biyectivo 26 y viceversa.

Entrada

En la primera línea, un entero \(T\) que indica el tipo de conversión que vas a hacer (0 = de decimal a cadena, 1 = de cadena a decimal).

En la segunda línea, un entero \(X\) o una cadena \(S\), dependiendo del tipo de conversión.

Salida

La cadena \(S\) que representa a \(X\) si la conversión es de tipo 0, o el entero \(X\) que representa a la cadena \(S\) si la conversión es de tipo 1.

Límites

Límite de tiempoNo especificado en la fuente
Límite de memoriaNo especificado en la fuente
Puntaje total100 puntos
Rango de \(X\)\(1 \le X < 2^{63}\)
Longitud de \(S\)\(|S| < 15\)

Subtareas

#PuntosRestricciones adicionales
150\(T = 0\): de decimal a cadena (el 50% de los casos).
250\(T = 1\): de cadena a decimal (el 50% restante).

El puntaje se reparte por la proporción de casos: la mitad evalúa la conversión \(T=0\) y la otra mitad la conversión \(T=1\).

Ejemplos

Entrada

1
AZ

Salida

52

\(T=1\) (cadena → decimal): AZ vale \(1\cdot 26 + 26 = 52\).

Entrada

0
10487

Salida

OMI

\(T=0\) (decimal → cadena): el entero \(10487\) se escribe como OMI en biyectivo base 26.

Entrada

1
ABC

Salida

731

\(T=1\): ABC vale \(((1)\cdot 26 + 2)\cdot 26 + 3 = 731\).

Resumen

Hay que hacer dos conversiones inversas en numeración biyectiva base 26 (la de las columnas de una hoja de cálculo: A=1, …, Z=26, AA=27, …). Si \(T=0\) te dan un entero \(X\) y debes imprimir su representación en letras; si \(T=1\) te dan una cadena \(S\) de letras mayúsculas y debes imprimir el entero que representa.

Lo que importa de los límites \(X\) llega casi a \(2^{63}\): el valor cabe en un entero de 64 bits (su máximo, \(2^{63}-1\), es justo el tope de long long con signo). La cadena tiene menos de 15 letras. No hay nada que optimizar: ambas conversiones son lineales en el largo de la respuesta.

Pistas

Pista 1 — ¿Qué tiene de raro esta "base 26"?

¿Ya notaste que NO es la base 26 común de la escuela? En una base 26 normal los dígitos van de 0 a 25 y existe el dígito cero; aquí los "dígitos" van de 1 a 26 (A..Z) y no hay ningún símbolo para el 0.

¿Qué consecuencia tiene que no exista el cero? Fíjate que después de Z (=26) no viene "una nueva cifra seguida de 0", sino AA (=27). ¿Cómo cambia eso el algoritmo de "dividir entre la base" que usarías normalmente?

Pista 2 — El sentido fácil: de cadena a entero (\(T=1\))

¿Cómo evalúas un número escrito en una base posicional? Si cada letra vale \(v(c) = c - \text{'A'} + 1\) (es decir, A=1, …, Z=26), ¿no es el valor simplemente el esquema de Horner?

Recorre la cadena de izquierda a derecha y acumula \(x \leftarrow x\cdot 26 + v(c)\). ¿Te sale \(52\) con AZ y \(731\) con ABC? Aquí el "cero faltante" no estorba: nunca lo necesitas, porque cada letra ya aporta de 1 a 26.

Pista 3 — El sentido difícil: de entero a cadena (\(T=0\))

Para pasar a una base normal harías repetidamente \(d = X \bmod 26\) y \(X \leftarrow X/26\). Pero aquí el residuo válido debe ser 1..26, no 0..25. ¿Qué pasa cuando \(X\) es múltiplo de 26 (el residuo "normal" daría 0)?

¿Y si restas 1 a \(X\) antes de tomar el residuo? Con \(X \leftarrow X-1\), el residuo \(X\bmod 26\) cae en 0..25, le sumas 1 para obtener la letra (A..Z) y luego \(X \leftarrow X/26\). Las letras salen de la menos significativa a la más significativa, así que al final, ¿qué tienes que hacer con la cadena?

Pista 4 — Cuidado con los límites

\(X\) puede llegar a \(2^{63}-1 = 9\,223\,372\,036\,854\,775\,807\). ¿Tu tipo entero aguanta ese tope? Un entero de 32 bits se desborda; usa 64 bits (con signo basta, o sin signo para mayor margen).

En el sentido \(T=1\), como \(|S|<15\) y el resultado cabe en \(< 2^{63}\), el acumulado nunca se sale del rango de 64 bits. ¿Lo verificaste con el caso extremo \(X = 2^{63}-1\)?

Pista final — Solución completa (100 puntos)

Idea. Son dos rutinas que son inversas la una de la otra. Lees \(T\); si es 1 decodificas (cadena → entero) y si es 0 codificas (entero → cadena).

Decodificar (\(T=1\)). Es una evaluación posicional en base 26 con dígitos 1..26 (Horner): recorres la cadena de izquierda a derecha y haces \(x \leftarrow x\cdot 26 + v(c)\), donde \(v(c)=c-\text{'A'}+1\). Como cada letra aporta entre 1 y 26, la ausencia de dígito cero no causa problema.

Codificar (\(T=0\)) — el truco biyectivo. En una base normal, para sacar el dígito menos significativo usarías \(d=X\bmod 26 \in \{0,\dots,25\}\). Pero aquí los dígitos son \(\{1,\dots,26\}\): no existe el 0. La solución es correr el rango restando 1 antes de cada paso. En cada iteración haces \(X \leftarrow X-1\), tomas \(d = X\bmod 26 \in \{0,\dots,25\}\) (que se traduce a la letra \(\text{'A'}+d\)) y luego \(X \leftarrow \lfloor X/26 \rfloor\). Repites mientras \(X>0\). Las letras se generan de la menos a la más significativa, así que al final inviertes la cadena.

Por qué es correcto. Para representar \(X\) con la cifra menos significativa \(a\in\{1,\dots,26\}\) y un "resto" \(q\) (lo que queda hacia la izquierda), se cumple \(X = 26\,q + a\). Despejando con \(a\) en \(\{1,\dots,26\}\): \(a = ((X-1)\bmod 26)+1\) y \(q = \lfloor (X-1)/26 \rfloor\). Eso es exactamente lo que hace el ajuste "\(X-1\)". Como la numeración biyectiva representa cada entero positivo de una sola forma, el proceso termina y produce la única cadena válida.

Pseudocódigo 
funcion Decodificar(S):          // T = 1  (cadena -> entero)
    x = 0
    para cada caracter c en S (de izquierda a derecha):
        d = (c - 'A') + 1        // A->1, ..., Z->26
        x = x * 26 + d
    regresa x                    // entero de 64 bits

funcion Codificar(X):            // T = 0  (entero -> cadena)
    S = cadena vacia
    mientras X > 0:
        X = X - 1                // corre el rango a 0..25 (no hay digito 0)
        d = X mod 26             // 0..25
        agrega ('A' + d) al final de S
        X = X / 26               // division entera
    invierte S
    regresa S

funcion Principal():
    lee T
    si T == 0:
        lee X ; imprime Codificar(X)
    sino:
        lee S ; imprime Decodificar(S)

Complejidad: \(O(|S|)\) en tiempo y memoria, donde \(|S|\le 14\) (a lo más unas 14 letras, o un solo entero). En la práctica es \(O(1)\).
