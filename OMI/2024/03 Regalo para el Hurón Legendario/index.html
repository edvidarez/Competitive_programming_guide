Problema 03 — Regalo para el Hurón Legendario

Enunciado oficial: statement.md · Practica en línea: omegaUp

El problema

Se acerca el cumpleaños del Hurón Legendario. Tú, como su buen amigo, quieres regalarle uno de sus muchos objetos favoritos: un arreglo.

Los gustos del Hurón son un poco complicados. El arreglo que le regales al Hurón debe cumplir lo siguiente:

  • Debe tener exactamente \(N\) elementos.
  • Cada elemento debe ser un valor entre \(1\) y \(M\).
  • No debe tener subarreglos prohibidos. En el siguiente párrafo se explica qué es un subarreglo prohibido.

El Hurón tiene una matriz \(V\) de tamaño \(N \times M\) que contiene valores enteros. Para algún subarreglo \(S\) del arreglo, sea \(S_{largo}\) el tamaño del subarreglo y \(S_{max}\) el elemento máximo en ese subarreglo. El subarreglo es prohibido si la celda \(V[S_{largo}][S_{max}] = -1\). Recuerda que un subarreglo es cualquier sección contigua del arreglo, incluyendo el arreglo completo. Los arreglos \(\{2, 3\}\) y \(\{3, 4\}\) son subarreglos de \(\{1, 2, 3, 4\}\), pero \(\{1, 3, 4\}\) no.

Además, el Hurón califica sus regalos. Si le regalas un arreglo que cumpla con sus requisitos, el Hurón le asignará un valor de la siguiente forma:

  • Para cada uno de los subarreglos del arreglo, el Hurón obtendrá su \(S_{largo_i}\) y \(S_{max_i}\).
  • Luego calculará la suma de \(V[S_{largo_i}][S_{max_i}]\) para todos los subarreglos. La suma total será el valor del arreglo.

Como aprecias mucho al Hurón, quieres darle todos los posibles arreglos que le gusten. Para esto, necesitas contar cuántos arreglos diferentes existen que cumplen sus requisitos y calcular la suma total de los valores de todos esos arreglos.

Lo que se pide

Dados \(N\), \(M\) y la matriz \(V\) de tamaño \(N \times M\), encuentra:

  • La cantidad de arreglos diferentes que le gustan al Hurón. Como este número puede ser muy grande, imprímelo módulo \(10^9+7\).
  • La suma total de los valores de todos los arreglos que le gustan al Hurón. Como este número puede ser muy grande, imprímelo módulo \(10^9+7\).

Entrada

  • La primera línea de la entrada contiene los enteros \(N\) y \(M\).
  • Las siguientes \(N\) líneas contienen \(M\) enteros cada una. Los enteros de la línea \(i+1\) son \(V[i][1], V[i][2], \dots, V[i][M]\) (los valores de la \(i\)-ésima fila de la matriz \(V\)).

Salida

Imprime dos enteros separados por un espacio: la cantidad de arreglos diferentes que cumplen con los requisitos del Hurón Legendario y la suma de los valores de todos esos arreglos, ambos módulo \(10^9+7\).

Ejemplo 1

Entrada

2 3
1 1 1
1 1 1

Salida

9 27

Existen \(9\) arreglos que cumplen los requisitos del Hurón: \([1, 1]\), \([1, 2]\), \([1, 3]\), \([2, 1]\), \([2, 2]\), \([2, 3]\), \([3, 1]\), \([3, 2]\) y \([3, 3]\). Cada uno tiene valor igual a \(3\), por lo tanto la suma total de los valores es \(27\).

Tomando como ejemplo el arreglo \([1, 3]\):

  • Se toman todos sus subarreglos; en este caso serían \(\{1\}\), \(\{3\}\) y \(\{1, 3\}\).
  • El primer subarreglo tiene \(largo = 1\) y elemento \(maximo = 1\); revisando la tabla \(V[1][1] = 1\), por lo que ese subarreglo vale \(1\).
  • El segundo subarreglo tiene \(largo = 1\) y elemento \(maximo = 3\); revisando la tabla \(V[1][3] = 1\), por lo que ese subarreglo vale \(1\).
  • El último subarreglo tiene \(largo = 2\) y elemento \(maximo = 3\); revisando la tabla \(V[2][3] = 1\), por lo que ese subarreglo vale \(1\).
  • El valor final del arreglo es la suma de todos sus subarreglos: \(res = 1 + 1 + 1 = 3\).

Ejemplo 2

Entrada

2 2
2 4
-1 1

Salida

3 23

Aquí \(V[2][1] = -1\): cualquier subarreglo de largo \(2\) cuyo máximo sea \(1\) está prohibido, por lo que el arreglo \([1,1]\) queda descartado. Los \(3\) arreglos válidos son \([1,2]\) (valor \(7\)), \([2,1]\) (valor \(7\)) y \([2,2]\) (valor \(9\)); su suma es \(23\).

Etiquetas Programación dinámica (DP) Combinatoria Conteo Sumas acumuladas

Resumen

Cuenta los arreglos de largo \(N\) con valores en \([1, M]\) tales que ningún subarreglo contiguo (de largo \(L\) y máximo \(v\)) caiga en una celda prohibida \(V[L][v] = -1\). Para cada arreglo válido, su "valor" es la suma sobre todos sus subarreglos de \(V[L][v]\). Reporta dos cosas módulo \(10^9+7\): cuántos arreglos válidos hay y la suma de los valores de todos ellos.

Lo que importa de los límites \(N, M \le 400\) pide algo polinómico, alrededor de \(O(N^2 M)\). Las subtareas son una escalera: \(V\equiv 1\) admite fórmula cerrada; \(N,M\le 6\) admite fuerza bruta; \(V\in\{-1,1\}\) elimina la parte de la suma y deja solo contar; lo demás afina la DP y los acumulados.

Pistas

Pista 1 — ¿Cómo modelarlo?

¿Notaste que la condición de "prohibido" de un subarreglo depende solo de dos números: su largo y su máximo? ¿Y que el problema en realidad pide dos cosas casi independientes?

Conviene separar mentalmente: (a) contar cuántos arreglos son válidos y (b) sumar el valor de todos los válidos. Ambas comparten la misma estructura de conteo; la (b) solo "carga" además el costo \(V[L][v]\) de cada subarreglo. Empieza por entender bien (a).

Pista 2 — Empieza por lo chico (subtareas 1 y 2)

Si todas las celdas valen \(1\) (subtarea 1), ¿hay algún arreglo prohibido? ¿Cuántos subarreglos tiene un arreglo de largo \(N\)?

Con \(V\equiv 1\) nada está prohibido: hay \(M^N\) arreglos, cada uno con \(\frac{N(N+1)}{2}\) subarreglos que aportan \(1\) cada uno. La respuesta es \(M^N\) y \(M^N \cdot \frac{N(N+1)}{2}\). Para \(N,M\le 6\) (subtarea 2) simplemente genera los \(M^N\) arreglos y verifica a mano sus subarreglos.

Esas dos subtareas dan \(15\) puntos casi gratis. ¿Qué patrón te sugieren para el caso general?

Pista 3 — La reducción clave de "prohibido"

Si un subarreglo válido tiene largo \(L\) y máximo \(v\), el valor \(v\) aparece en alguna posición \(p\) dentro de él. ¿Qué pasa con los subarreglos más chicos que también contienen a \(p\)?

Partiendo de la celda \(p\) (que vale \(v\)) y creciendo, obtienes subarreglos cuyo máximo sigue siendo \(v\) con largos \(1, 2, \dots, L\). Para que el arreglo sea válido necesitas que \(V[1][v], V[2][v], \dots, V[L][v]\) sean todos distintos de \(-1\).

Define \(\text{lim}[v]\) = el mayor \(L\) tal que el prefijo \(V[1..L][v]\) no contiene ningún \(-1\). Entonces la validez se vuelve una regla simple: todo subarreglo con máximo \(v\) debe tener largo \(\le \text{lim}[v]\). ¡Ya no hace falta mirar la matriz celda por celda!

Pista 4 — La DP de conteo (subtareas 3–5)

¿Y si construyes los arreglos "por valor máximo", agregando los valores de menor a mayor? ¿Cómo cuentas sin repetir?

Define \(\text{cnt}[i][c]\) = número de arreglos válidos de largo \(i\) cuyos valores están en \([1, c]\). Para pasar de \(c-1\) a \(c\): o no usas el valor \(c\) (eso es \(\text{cnt}[i][c-1]\)), o el máximo del arreglo es exactamente \(c\). En el segundo caso, fija la primera posición \(k\) donde aparece \(c\): el prefijo \([1..k-1]\) usa valores \(\le c-1\) y el sufijo \([k+1..i]\) usa valores \(\le c\). Eso cuenta cada arreglo exactamente una vez.

¿Cuándo puede el máximo ser exactamente \(c\)? El arreglo completo es un subarreglo de largo \(i\) y máximo \(c\): por la Pista 3 solo se permite si \(i \le \text{lim}[c]\).

La versión directa es \(O(N^2 M^2)\); manteniendo acumulados sobre los valores menores que el máximo se baja a \(O(N^2 M)\), que es lo que pide el límite.

Pista final — Solución completa (100 puntos)

Idea. Reduce la validez con \(\text{lim}[c]\) (Pista 3) y lleva dos tablas con la misma descomposición:

  • \(\text{cnt}[i][c]\): cantidad de arreglos válidos de largo \(i\) con valores en \([1, c]\).
  • \(\text{val}[i][c]\): suma de los valores de todos esos arreglos.

Descomposición. Un arreglo de largo \(i\) con valores en \([1, c]\) o tiene máximo \(< c\) (cae en la columna \(c-1\)) o tiene máximo exactamente \(c\). Para el segundo caso, sea \(k\) la primera aparición de \(c\): se parte en prefijo \(P\) (largo \(k-1\), valores \(\le c-1\)), la celda \(c\) en la posición \(k\), y sufijo \(Q\) (largo \(i-k\), valores \(\le c\)). Cada arreglo con máximo \(c\) se cuenta una vez, y solo es válido si \(i \le \text{lim}[c]\).

Conteo. Se multiplican las opciones de prefijo y sufijo: \(\text{cnt}[i][c] \mathrel{+}= \text{cnt}[k-1][c-1] \cdot \text{cnt}[i-k][c]\) (con la convención de que un lado vacío cuenta como \(1\)).

Suma de valores. Al pegar \(P + [c] + Q\), cada subarreglo cae en exactamente una de tres categorías disjuntas:

  • Dentro de \(P\): aportan \(\text{val}(P)\), repetido por cada sufijo \(\Rightarrow \text{val}[k-1][c-1]\cdot \text{cnt}[i-k][c]\).
  • Dentro de \(Q\): aportan \(\text{val}(Q)\), repetido por cada prefijo \(\Rightarrow \text{cnt}[k-1][c-1]\cdot \text{val}[i-k][c]\).
  • Que cruzan por la posición \(k\): su máximo es siempre \(c\) (la celda \(k\) es el máximo), así que su costo no depende de los valores concretos de \(P\) y \(Q\); por eso se multiplican por \(\text{cnt}[k-1][c-1]\cdot \text{cnt}[i-k][c]\).

Costo de los subarreglos que cruzan. Llamémoslo \(\text{cruz}(k, i)\): la suma de \(V[\text{largo}][c]\) sobre todos los subarreglos \([l, r]\) con \(l \le k \le r\), \(1 \le l\) y \(r \le i\). Fijando \(a = k - l + 1\) (cantidad de posiciones del lado izquierdo, incluida \(k\)), el largo recorre desde \(a\) hasta \(a + (i-k)\). Con un prefijo de costos \(P_c[t] = \sum_{\ell=1}^{t} V[\ell][c]\):

\[ \text{cruz}(k, i) = \sum_{a=1}^{k} \big( P_c[a + i - k] - P_c[a-1] \big). \]

Como \(i \le \text{lim}[c]\), todos los largos consultados son \(\le \text{lim}[c]\), es decir nunca tocan una celda \(-1\). Estos valores se precalculan por columna \(c\) en \(O(N^2)\) con una recurrencia acumulada.

Pseudocódigo 
leer N, M, V[1..N][1..M]
para c = 1..M:  lim[c] = N
para i = 1..N, j = 1..M:
    si V[i][j] == -1:  lim[j] = min(lim[j], i-1)   // mayor prefijo sin -1

// bases (arreglo vacío)
para c = 0..M:  cnt[0][c] = 1 ;  val[0][c] = 0
para i = 1..N:  cnt[i][0] = 0 ;  val[i][0] = 0

para c = 1..M:
    // prefijos de costo de la columna c (las celdas -1 nunca se usan en rango válido)
    P[0] = 0
    para t = 1..N:  P[t] = (P[t-1] + max(V[t][c], 0)) mod p
    // D[l][r] = sum_{a=1..l} (P[a+r-1] - P[a-1]) ; válido cuando a+r-1 <= N
    para r = 1..N:  D[0][r] = 0
    para l = 1..N, r con l+r-1 <= N:
        D[l][r] = ( D[l-1][r] + (P[l+r-1] - P[l-1]) ) mod p
    // entonces  cruz(k, i) = D[k][ i-k+1 ]

    para i = 1..N:
        cnt[i][c] = cnt[i][c-1]
        val[i][c] = val[i][c-1]
        si lim[c] >= i:                  // un arreglo de largo i con máximo c es válido
            para k = 1..i:               // k = primera aparición del máximo c
                ic = cnt[k-1][c-1] ; iv = val[k-1][c-1]   // prefijo (vacío -> 1, 0)
                dc = cnt[i-k][c]   ; dv = val[i-k][c]     // sufijo  (vacío -> 1, 0)
                cnt[i][c] += ic * dc
                val[i][c] += iv*dc + ic*dv + D[k][i-k+1] * ic * dc
        // todas las operaciones bajo módulo p = 1e9+7

imprimir  cnt[N][M],  val[N][M]

Cómo sube de subtarea en subtarea. Con \(V\equiv 1\) (subtarea 1) basta la fórmula cerrada. Con \(N,M\le 6\) (subtarea 2), fuerza bruta. Las subtareas 3–4 (\(V\in\{-1,1\}\)) solo necesitan la tabla de conteo (la suma de valores se vuelve trivial) y \(\text{lim}[c]\). Las subtareas 5–6 agregan la tabla \(\text{val}\) y el costo de los subarreglos que cruzan; la versión con acumulados llega a \(100\).

Complejidad: \(O(N^2 M)\) en tiempo y \(O(N^2 + N M)\) en memoria (la tabla \(D\) por columna es \(O(N^2)\) y se reutiliza entre columnas).
