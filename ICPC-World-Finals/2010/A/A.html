ICPC World Finals · 2010

Problema A — APL Lives!

PDF original del concurso: ../problems.pdf

El problema

APL es un lenguaje de programación orientado a arreglos que usa una notación inventada por Ken Iverson en 1957. En este problema solo consideramos un pequeño subconjunto del lenguaje, al que llamamos apl (es decir, small APL).

Cada expresión de apl aparece sola en su propia línea y cada expresión tiene un valor, que se muestra inmediatamente después de teclear la expresión. Los operadores de apl no tienen precedencia como los de C, C++ o Java; en su lugar se aplican de derecha a izquierda. Sin embargo, se pueden usar paréntesis para controlar el orden de evaluación. De forma análoga, los operandos de los operadores binarios se evalúan de derecha a izquierda. Estos son algunos ejemplos de expresiones de apl:

Ejemplos del lenguaje (traducidos)

  • var = 1 2 3 — Guarda el vector 1 2 3 en var, reemplazando su valor anterior. El valor de la expresión es 1 2 3. El operando izquierdo del operador = debe ser una variable.
  • var + 4 — Muestra el valor de var sumando 4 a cada uno de sus elementos (resultado: 5 6 7); la versión guardada de var no se modifica.
  • - / var — Muestra el valor de var como si se hubiera insertado un operador - entre cada par de elementos de cada renglón (resultado: 2). Si var tiene dos dimensiones, el resultado es un vector; si tiene tres dimensiones, el resultado es un arreglo bidimensional. * / y + / se comportan de forma análoga.
  • iota 5 — Genera un vector con los valores 1 2 3 4 5.
  • 2 2 rho 1 2 3 4 — Reorganiza el vector 1 2 3 4 en un arreglo de \(2\times 2\); el 1 y el 2 quedan en el primer renglón y el 3 y el 4 en el segundo.
  • 2 2 rho 1 2 3 4 5 6 — Mismo resultado que el anterior (los elementos sobrantes se ignoran).
  • 2 3 rho 1 2 3 4 — Otra reorganización, que produce un primer renglón 1 2 3 y un segundo renglón 4 1 2; si el argumento derecho no tiene suficientes elementos, estos se reutilizan desde el principio, en orden por renglones (row-major).
  • 2 drop iota 5 — Resultado: 3 4 5. Elimina los dos primeros elementos de iota 5.
  • 1 2 * 3 4 — Resultado: 3 8. Ilustra la multiplicación elemento a elemento. Los operandos deben ser conformables: o tienen la misma forma, o al menos uno es un vector de un solo elemento (ver el segundo ejemplo).
  • ( ( a = 1 ) drop 1 2 3 ) - 5 — Resultado: -3 -2. Ilustra el uso de paréntesis.
  • a + ( a = 5 ) + a + ( a = 6 ) — Resultado: 22. Ilustra el orden de evaluación.

En este problema debes escribir un intérprete para apl. Los enteros de la entrada son no negativos y menores que \(10^4\). Todos los valores enteros calculados (incluyendo los valores intermedios) tienen valor absoluto menor que \(10^4\). El número de elementos de cualquier matriz es siempre \(\le 10^4\). Los nombres de variable constan de una a tres letras minúsculas, y los nombres iota, rho y drop siempre se interpretan como operadores. Exactamente un espacio separa los elementos de los enunciados (constantes, variables, operadores y paréntesis).

Las constantes de la entrada son vectores. Todos los valores intermedios son arreglos de una, dos o tres dimensiones con dimensiones positivas. Esto restringe los rangos de algunos operandos: 2 0 rho 1 2 3, 2 3 2 1 rho 5 y 3 drop iota 3 son ilegales. Los únicos operadores aritméticos disponibles son + (suma), - (resta) y * (multiplicación). Sus operandos son conformables como se ilustra en los ejemplos. Observa que 1 1 rho 1 y 1 rho 1 tienen formas distintas. El operando de iota evalúa a un vector positivo de un solo elemento. El operando izquierdo de drop evalúa a un vector no negativo de un solo elemento, y su operando derecho evalúa a un vector. Ambos operandos de rho evalúan a vectores.

Entrada

La entrada contiene varios casos de prueba, cada uno solo en su propia línea. Los valores de las variables asignadas en un caso de prueba están disponibles para ser usados en los casos de prueba siguientes. Ninguna expresión excede los 80 caracteres de longitud, incluyendo los espacios. Ningún caso de prueba produce un resultado inválido (por ejemplo, un vector vacío).

El último caso de prueba va seguido de una línea que contiene el único carácter #.

Salida

Para cada caso de prueba, muestra una línea con el número de caso y la línea de entrada. Luego, empezando en la siguiente línea, muestra el resultado de evaluar la expresión. Los vectores se muestran como una sola línea de enteros; los arreglos de \(m\times n\) se muestran como \(m\) líneas de \(n\) valores; y los arreglos de \(m\times n\times p\) se muestran como \(m\) arreglos de tamaño \(n\times p\), con una línea en blanco separando los arreglos \(n\times p\). Los valores de una misma línea deben separarse por espacios en blanco, pero no necesitan quedar alineados en columnas.

Resumen

Te dan líneas con expresiones de un mini-APL: constantes (vectores de enteros), variables (estado persistente entre casos), aritmética elemento a elemento (+ - *), asignación (=), reducción (op /), iota, reshape (rho) y drop. Para cada línea hay que evaluar la expresión respetando que todo se aplica de derecha a izquierda (con paréntesis para forzar el orden) y los operandos también se evalúan de derecha a izquierda. Hay que imprimir el valor resultante como vector / arreglo 2-D / arreglo 3-D.

Lo que importa de los límites Tamaños diminutos (\(\le 80\) caracteres por línea, \(\le 10^4\) elementos por arreglo, valores \(< 10^4\)). No hay reto de eficiencia: el reto es implementar bien la semántica y el orden de evaluación.

Pistas

Pista 1 — ¿Cómo represento un "valor"?

¿Ya pensaste en que todo valor (constante, resultado de rho, de una reducción, etc.) es lo mismo: un arreglo de 1, 2 o 3 dimensiones?

Te conviene un único tipo: una lista de enteros aplanada (orden row-major) más un vector de forma shape de 1 a 3 enteros. Un vector tiene shape=[n]; una matriz shape=[m,n]; un cubo shape=[m,n,p]. Casi todas las operaciones se vuelven triviales si trabajas sobre los datos aplanados y solo cuidas la shape.

Pista 2 — ¿Cómo parseo "de derecha a izquierda" sin precedencia?

Sin precedencia y todo a derecha, ¿qué significa exactamente el operando izquierdo y el derecho de un operador binario?

Regla de APL: para el primer operador que encuentres al recorrer la línea de izquierda a derecha (al nivel superior, es decir, fuera de paréntesis), su operando derecho es TODO lo que está a su derecha (se evalúa recursivamente), y su operando izquierdo es TODO lo que está a su izquierda (que, por construcción, no contiene operadores de nivel superior: es un literal de vector, una variable o una expresión entre paréntesis). Eso te da asociatividad a la derecha de forma natural.

Pista 3 — Los dos operadores que rompen el patrón

¿Qué pasa con - / var (reducción) y con a = ... (asignación)? ¿Encajan en "binario normal"?

Dos casos especiales:

  • Reducción op /: cuando el operador (+ - *) va seguido del token /, no hay operando izquierdo; reduces el operando derecho a lo largo del último eje, de derecha a izquierda (¡por eso - / 1 2 3 da \(1-(2-3)=2\), no \(-4\)!).
  • Asignación v =: el operando izquierdo es el nombre de la variable (no su valor). Evalúas el derecho, lo guardas y lo devuelves como valor de la expresión.
Pista 4 — El orden de evaluación de los operandos importa

En a + ( a = 5 ) + a + ( a = 6 ), ¿por qué da 22? ¿En qué orden tocas los operandos?

Para un binario, evalúa primero el operando derecho y luego el izquierdo, porque las asignaciones tienen efecto lateral sobre las variables. Si evaluaras el izquierdo primero leerías valores viejos. Con derecho→izquierdo, el a de la izquierda lee el último valor asignado a su derecha.

Pista final — Solución completa (upsolving)

Idea general. Es un problema de parsing + simulación. Tokeniza por espacios y escribe un evaluador recursivo sobre una sublista de tokens tok[l..r). Un valor es (datos[], shape[]) con 1 ≤ |shape| ≤ 3, datos en orden row-major.

Evaluador eval(l, r). Recorre de l a r llevando un contador de profundidad de paréntesis. El primer token a profundidad 0 que sea operador determina la operación:

  • Si no hay ningún operador a nivel 0: el segmento es un átomo. Si está envuelto en paréntesis que abarcan todo, quítalos y recurre. Si es un solo nombre de variable, devuelve una copia de su valor. Si son puros números, devuelve el vector con esos números, shape=[cantidad].
  • Reducción (operador + - * seguido de /): evalúa el operando derecho (tokens tras el /) y reduce el último eje de derecha a izquierda.
  • Asignación (=): el token a su izquierda es el nombre; evalúa la derecha, guárdala en la tabla de variables y devuélvela.
  • Aritmético / rho / drop (binarios) e iota (unario): evalúa el operando derecho primero (todo lo de la derecha), luego el izquierdo (todo lo de la izquierda), y aplica.

Semántica de cada operador.

  • iota n: n es vector de 1 elemento positivo; devuelve [1..n], shape [n].
  • k drop v: k 1 elemento no negativo, v vector; devuelve v sin sus k primeros, shape [len-k].
  • s rho v: s es la forma (1–3 enteros), v un vector; llena prod(s) elementos en row-major reutilizando v cíclicamente (índice i mod len(v)).
  • x op y con op∈{+,-,*}: si misma forma, opera elemento a elemento; si uno tiene exactamente 1 elemento, hace broadcast de ese escalar sobre el otro y el resultado toma la forma del otro.
  • op / x: reduce el último eje. El arreglo tiene shape=[d1,..,dk]; agrupa los datos en bloques contiguos de tamaño dk y cada bloque [a0,a1,...,a(dk-1)] se colapsa a a0 op (a1 op (... op a(dk-1))) (asociado a la derecha). La forma resultante es shape sin su último eje (un vector se reduce a un escalar).

Impresión. Con shape [n]: una línea con los n valores. Con [m,n]: m líneas de n valores. Con [m,n,p]: m bloques de n×p, separados por una línea en blanco. (En la salida de ejemplo cada línea de valores empieza con un espacio inicial; replica ese formato.) Antes, imprime Case N: seguido de la línea de entrada tal cual.

Por qué es correcto. La regla "primer operador a nivel 0 toma toda su izquierda como operando izquierdo y toda su derecha como operando derecho" reproduce exactamente la asociatividad a la derecha sin precedencia de APL; los paréntesis se respetan porque solo miramos operadores a profundidad 0. Evaluar el operando derecho antes que el izquierdo reproduce el orden derecha→izquierda de los efectos laterales (verificado con el caso 10 = 22). La reducción asociada a la derecha reproduce - / 1 2 3 = 2.

Pseudocódigo 
vars = {}                       // nombre -> Valor(datos, shape)

func eval(tok, l, r) -> Valor:
    // 1) ¿hay operador a nivel 0?
    depth = 0; opPos = -1
    for i in [l, r):
        if tok[i]=="(": depth++
        elif tok[i]==")": depth--
        elif depth==0 and esOperador(tok[i]) and opPos==-1:
            opPos = i
    if opPos == -1:                       // átomo
        if tok[l]=="(" y empareja con tok[r-1]:
            return eval(tok, l+1, r-1)
        if esVariable(tok[l]) y r-l==1:
            return copia(vars[tok[l]])
        return Valor(datos=[int(t) for t in tok[l..r)], shape=[r-l])

    t = tok[opPos]
    // 2) reducción:  op /
    if t in {+,-,*} y opPos+1 < r y tok[opPos+1]=="/":
        x = eval(tok, opPos+2, r)
        return reduceUltimoEje(t, x)
    // 3) asignación
    if t == "=":
        val = eval(tok, opPos+1, r)       // derecho
        vars[ tok[opPos-1] ] = copia(val) // izquierdo = nombre
        return val
    // 4) unario iota
    if t == "iota":
        y = eval(tok, opPos+1, r)
        return iota(y[0])
    // 5) binarios: rho, drop, + - *   (¡derecho primero!)
    y = eval(tok, opPos+1, r)             // operando derecho
    x = eval(tok, l, opPos)               // operando izquierdo
    switch t:
      case "rho":  return reshape(x.datos, y.datos)
      case "drop": return Valor(y.datos[x[0]:], shape=[len(y.datos)-x[0]])
      case "+","-","*": return aritmetica(t, x, y)   // con broadcast

func reduceUltimoEje(op, A):
    d = A.shape.last
    out = []
    for cada bloque B de tamaño d en A.datos (row-major):
        acc = B[d-1]
        for i = d-2 downto 0: acc = op(B[i], acc)   // derecha->izquierda
        out.push(acc)
    nuevaShape = A.shape sin el ultimo (o escalar si queda vacio)
    return Valor(out, nuevaShape)

func aritmetica(op, x, y):
    if len(x.datos)==1: aplica escalar x sobre y, shape = y.shape
    elif len(y.datos)==1: aplica escalar y sobre x, shape = x.shape
    else: elemento a elemento, shape = x.shape   // misma forma

Complejidad: con cada arreglo de \(\le 10^4\) elementos y líneas de \(\le 80\) caracteres (a lo más \(\sim 40\) tokens), cada evaluación es \(O(\text{tokens}^2)\) en el peor caso por la búsqueda lineal de operadores anidados, más \(O(\text{elementos})\) por operación. En la práctica es trivial: muy por debajo de cualquier límite. Memoria \(O(\text{elementos})\) por valor.
