ICPC World Finals · 2023

Problema C — Three Kinds of Dice

PDF original del concurso: ../problems.pdf

El problema

¡Mira cómo ruedan! Cuenta una famosa anécdota que Warren Buffett retó una vez a Bill Gates a un sencillo juego de dados. Tenía tres dados; el primer jugador podía examinarlos y elegir uno de los tres. El segundo jugador elegía luego uno de los dados restantes, y ambos lanzaban su dado uno contra el otro, buscando el número más alto. Warren ofreció dejar que Bill eligiera primero, pero esto le dio mala espina a Bill y prefirió ir en segundo lugar. Resultó ser una decisión sabia: estos eran dados intransitivos. El primer dado tenía ventaja contra el segundo, el segundo tenía ventaja contra el tercero, ¡pero el primero no tenía ventaja contra el tercero!

Para formalizarlo: definimos un «dado» como cualquier forma con al menos una cara, tal que cada cara muestra un entero positivo. Al lanzar un dado, se elige una de sus caras de manera uniforme al azar. Cuando dos dados se enfrentan, el dado cuya cara elegida muestra el número más alto gana 1 punto; si ambos números son iguales, cada dado gana \(\tfrac12\) punto. Para dados \(D\) y \(D'\), definimos \(\operatorname{score}(D, D')\) como el número esperado de puntos que \(D\) obtiene en un solo lanzamiento contra \(D'\). Si \(\operatorname{score}(D, D') > \tfrac12\), decimos que \(D\) tiene ventaja sobre \(D'\); si \(\operatorname{score}(D, D') = \tfrac12\), los dos dados están empatados.

Por ejemplo, si \(D\) es el primer dado del ejemplo de entrada y \(D'\) es el segundo, entonces \(\operatorname{score}(D, D') = \tfrac49\) y \(\operatorname{score}(D', D) = \tfrac59\), así que \(D'\) tiene ventaja sobre \(D\).

Te dan dos dados \(D_1\) y \(D_2\) tales que \(D_1\) tiene ventaja sobre \(D_2\). Quieres un tercer dado \(D_3\) que forme un trío intransitivo con los otros dos. Hay dos cantidades que calcular:

  • Entre todos los \(D_3\) que tienen ventaja sobre \(D_1\) o empatan con \(D_1\) (es decir, \(\operatorname{score}(D_3, D_1) \ge \tfrac12\)), calcula el menor valor posible de \(\operatorname{score}(D_3, D_2)\). Si ese valor es menor que \(\tfrac12\), ¡puedes armar un trío intransitivo!
  • De manera análoga, entre todos los \(D_3\) tales que \(D_2\) tiene ventaja sobre \(D_3\) o empata con \(D_3\) (es decir, \(\operatorname{score}(D_3, D_2) \le \tfrac12\)), calcula el mayor valor posible de \(\operatorname{score}(D_3, D_1)\).

Entrada

La entrada contiene dos líneas, cada una describiendo un dado. Uno de los dados (el primero o el segundo) tiene ventaja sobre el otro. El dado con la ventaja es \(D_1\) y el otro es \(D_2\).

El primer entero de cada línea da \(n\) (\(1 \le n \le 10^5\)), el número de caras del dado. Luego siguen \(n\) enteros \(f_i\) (\(1 \le f_i \le 10^9\) para cada \(1 \le i \le n\)), que dan el entero escrito en cada cara.

Salida

Imprime una línea que contenga el menor \(\operatorname{score}(D_3, D_2)\) y el mayor \(\operatorname{score}(D_3, D_1)\) bajo las condiciones de arriba. Los dos valores no tienen por qué usar el mismo dado \(D_3\). Tu respuesta debe tener un error absoluto de a lo más \(10^{-6}\).

Entrada

Salida

0.291666667 0.750000000

Aquí el segundo dado \(\{2,4,9\}\) tiene la ventaja, así que \(D_1=\{2,4,9\}\) y \(D_2=\{1,1,6,6,8,8\}\). Por ejemplo, el dado \(D_3=\{1,5,5,5\}\) empata con \(D_1\) (\(\operatorname{score}(D_3,D_1)=\tfrac12\)) y cumple \(\operatorname{score}(D_3,D_2)=0.291\overline{6} < \tfrac12\): como es menor que \(\tfrac12\), sí existe un trío intransitivo.

Ejemplo 2

Entrada

3 4 2 3

Salida

Aquí \(D_1=\{3,5,7,9\}\) domina con tanta fuerza que ningún \(D_3\) logra bajar de \(\tfrac12\): no se puede formar un trío intransitivo.

Resumen

Te dan dos dados \(D_1\) (con ventaja) y \(D_2\), cada uno como un multiconjunto de caras. Define \(\operatorname{score}(A,B)\) como la probabilidad de que una cara aleatoria de \(A\) gane a una de \(B\) (empate vale \(\tfrac12\)). Hay que calcular, sobre todos los dados posibles \(D_3\): (1) el mínimo de \(\operatorname{score}(D_3,D_2)\) sujeto a \(\operatorname{score}(D_3,D_1)\ge\tfrac12\); y (2) el máximo de \(\operatorname{score}(D_3,D_1)\) sujeto a \(\operatorname{score}(D_3,D_2)\le\tfrac12\).

Lo que importa de los límites \(n \le 10^5\) por dado y \(f_i \le 10^9\): hay que ordenar y usar aritmética exacta con enteros (\(O(n\log n)\)). Las funciones de puntaje solo cambian en los valores de las caras, así que solo hay \(O(n)\) «puntos candidatos».

Pistas

Pista 1 — ¿Qué es realmente un dado \(D_3\)?

Un dado puede tener cualquier cantidad de caras con cualquier entero positivo. ¿No es eso lo mismo que elegir una distribución de probabilidad sobre los enteros positivos?

Como puedes poner tantas caras como quieras, \(D_3\) equivale a una distribución \(p_v\) sobre valores \(v\) (con probabilidades racionales, pero eso basta para acercarse a cualquier valor). Olvídate de «caras»: piensa en «con qué probabilidad sale cada número».

Pista 2 — La observación clave: linealidad

Pista 3 — Del conjunto de puntos a una región convexa

Si cada \(D_3\) da el par \(\bigl(\operatorname{score}(D_3,D_1),\operatorname{score}(D_3,D_2)\bigr)\) como combinación convexa de los puntos \(P(v)\), ¿qué figura geométrica forman todos los pares alcanzables?

El conjunto de pares alcanzables es exactamente la envolvente convexa de los puntos \(\{P(v)\}\). Además, \(s_{D_1}\) y \(s_{D_2}\) solo cambian en los valores de las caras, así que solo hay \(O(n)\) puntos distintos: cada valor de cara (caso de empate), un entero dentro de cada hueco entre caras, los enteros por debajo del mínimo (si son \(\ge 1\)) y uno por encima del máximo.

Pista 4 — Reescribir las dos preguntas

Con el eje \(x = \operatorname{score}(D_3,D_1)\) y el eje \(y = \operatorname{score}(D_3,D_2)\), ¿cómo se ven las dos consultas sobre la región convexa?

Quedan así:

  • Respuesta 1 = mínimo de \(y\) sobre la envolvente con \(x \ge \tfrac12\) (la frontera inferior cortada por la recta vertical \(x=\tfrac12\)).
  • Respuesta 2 = máximo de \(x\) sobre la envolvente con \(y \le \tfrac12\) (la frontera derecha cortada por la recta horizontal \(y=\tfrac12\)).

Como \(s_{D_1}(v)\) y \(s_{D_2}(v)\) crecen ambas con \(v\), los puntos ya vienen ordenados de forma monótona, lo que hace muy directa la envolvente convexa.

Pista final — Solución completa (upsolving)

Idea. Un dado \(D_3\) es una distribución sobre valores. Por linealidad, el par \(\bigl(\operatorname{score}(D_3,D_1),\operatorname{score}(D_3,D_2)\bigr)\) es una combinación convexa de los puntos \(P(v)=\bigl(s_{D_1}(v),s_{D_2}(v)\bigr)\). El conjunto de pares alcanzables es la envolvente convexa de esos puntos. Las dos respuestas son optimizaciones lineales sobre esa envolvente.

Construcción de los puntos. Ordena las caras de \(D_1\) y de \(D_2\). Los puntajes \(s_{D_1}(v),s_{D_2}(v)\) solo cambian en los valores de cara, así que basta evaluar:

  • cada valor de cara \(w\) presente (donde aplica el medio punto por empate), y
  • un valor «representante» en cada hueco: justo debajo del mínimo de caras (solo si ese valor es \(\ge 1\)) y un valor estrictamente entre caras consecutivas, y uno por encima del máximo.

Las dos consultas. Construye la envolvente convexa (monotone chain). Luego:

  • Respuesta 1: recorre la frontera inferior (la que minimiza \(y\)). Es convexa en \(x\); el mínimo de \(y\) con \(x\ge\tfrac12\) está en el vértice óptimo o en el cruce de la frontera con \(x=\tfrac12\).
  • Respuesta 2: recorre la frontera derecha (la que maximiza \(x\)). El máximo de \(x\) con \(y\le\tfrac12\) está en un vértice o en el cruce de la frontera con \(y=\tfrac12\).

Por qué es correcto. Como el puntaje es lineal en la distribución, todo par alcanzable es una combinación convexa de los \(P(v)\) y, recíprocamente, cualquier combinación convexa se realiza con un dado de muchas caras. Optimizar una coordenada con una restricción lineal sobre un politopo convexo se logra siempre en su frontera (un vértice o un punto del borde sobre la recta de restricción), que es lo que recorremos. La verificación a mano contra ambos ejemplos da exactamente \(0.291\overline{6},\,0.75\) y \(0.5,\,0.5\).

Pseudocódigo 
leer D1, D2  // D1 = el que tiene ventaja (verificarlo con score)
ordenar caras de D1 y de D2

// 1) puntos candidatos P(v) = ( s_D1(v), s_D2(v) )
s_X(v) = ( (#caras de X < v) + (#caras de X == v)/2 ) / |X|
candidatos = { cada valor de cara de D1 ∪ D2 }
            ∪ { un entero en cada hueco entre caras consecutivas }
            ∪ { min_cara - 1  si min_cara > 1 }
            ∪ { max_cara + 1 }
puntos = [ ( s_D1(v), s_D2(v) ) para v en candidatos ]   // O(n) con binaria

// 2) envolvente convexa
H = envolvente_convexa(puntos)            // monotone chain, O(n log n)

// 3a) Respuesta1 = min y con x >= 1/2
ans1 = +inf
para cada arista (A,B) de H:
    para P in {A, B}: si P.x >= 1/2: ans1 = min(ans1, P.y)
    si la arista cruza x = 1/2:  ans1 = min(ans1, y_en(x=1/2))

// 3b) Respuesta2 = max x con y <= 1/2
ans2 = -inf
para cada arista (A,B) de H:
    para P in {A, B}: si P.y <= 1/2: ans2 = max(ans2, P.x)
    si la arista cruza y = 1/2:  ans2 = max(ans2, x_en(y=1/2))

imprimir ans1, ans2   // 9 decimales

Complejidad: \(O(n\log n)\) en tiempo (ordenar + búsquedas binarias + envolvente) y \(O(n)\) en memoria.

Truco de C++ (opcional) 
Para construir y comparar los puntos P(v) sin error de punto flotante,
guarda cada coordenada como fracción (numerador/denominador con
denominadores |D1|, |D2| <= 1e5). El producto cruzado de la envolvente
puede desbordar 64 bits, así que usa __int128 al multiplicar
numeradores en el cross product. Solo conviertes a double al imprimir.

El resto del algoritmo es independiente del lenguaje; este truco solo evita problemas de precisión en la envolvente.
