ICPC World Finals · 2014

Problema K — Surveillance

PDF original del concurso: ../problems.pdf

El problema

La International Corporation for Protection and Control (ICPC) desarrolla tecnología eficiente para, justamente, protección y control. Como es de esperarse, quieren tener su propio edificio central protegido y vigilado.

Visto desde arriba, el edificio central tiene la forma de un polígono convexo. Alrededor de él hay varios lugares adecuados donde se pueden instalar cámaras para monitorear el edificio. Cada cámara cubre cierto rango de lados del polígono (las paredes del edificio), dependiendo de su posición. La ICPC quiere minimizar el número de cámaras necesarias para cubrir todo el edificio.

Entrada

La entrada consiste en un solo caso de prueba. Su primera línea contiene dos enteros \(n\) y \(k\) (\(3 \le n \le 10^6\) y \(1 \le k \le 10^6\)), donde \(n\) es el número de paredes y \(k\) es el número de lugares posibles para instalar cámaras.

Cada una de las \(k\) líneas restantes contiene dos enteros \(a_i\) y \(b_i\) (\(1 \le a_i, b_i \le n\)). Estos enteros especifican qué paredes cubriría una cámara colocada en el \(i\)-ésimo lugar:

  • Si \(a_i \le b_i\), la cámara cubre cada pared \(j\) tal que \(a_i \le j \le b_i\).
  • Si \(a_i > b_i\), la cámara cubre cada pared \(j\) tal que \(a_i \le j \le n\) o bien \(1 \le j \le b_i\) (es decir, el rango le da la vuelta al polígono).

Salida

Muestra el número mínimo de cámaras que bastan para cubrir cada pared del edificio. Los rangos cubiertos por dos cámaras pueden traslaparse. Si el edificio no se puede cubrir, muestra impossible en su lugar.

Límites

Resumen

Tienes un círculo de \(n\) paredes numeradas \(1..n\) y \(k\) arcos (cada cámara cubre un rango de paredes consecutivas, posiblemente dándole la vuelta al círculo). Calcula el mínimo número de arcos que cubren las \(n\) paredes, o impossible si alguna pared no la cubre nadie.

Lo que importa de los límites \(n, k \le 10^6\) ⇒ hace falta algo cercano a \(O((n+k)\log n)\). Un greedy lineal por cada posible inicio sería \(O(n^2)\) y revienta.

Pistas

Pista 1 — ¿Qué tipo de problema es en realidad?

¿Ya notaste que el polígono es solo un pretexto? ¿Qué pasa si "desenrollas" las paredes \(1..n\) y piensas en cada cámara simplemente como un intervalo de enteros?

Olvida la geometría: tienes un círculo de \(n\) posiciones y un montón de intervalos (arcos). Quieres el mínimo de intervalos que cubren todo el círculo. Es el clásico minimum circular interval cover.

Pista 2 — La molestia del "círculo" y cómo quitarla

¿El que algunos arcos den la vuelta (\(a>b\)) te complica la vida? ¿Y si duplicas la recta para que el círculo se vuelva un segmento lineal?

Convierte cada cámara en un intervalo lineal: si \(a \le b\) es \([a,b]\); si \(a > b\) es \([a,\,b+n]\). Además, coloca una copia de cada intervalo desplazada \(+n\) (es decir \([a+n,\,b+n]\)), trabajando sobre el rango \(1..2n\). Así, cubrir el círculo equivale a, partiendo de alguna pared \(s\), cubrir el tramo lineal \([s,\,s+n-1]\).

Pista 3 — El greedy del "salto más lejano"

Fijado un punto que aún no cubres, ¿qué cámara conviene poner? ¿No es siempre la que, cubriéndolo, llegue lo más lejos posible?

Para cada posición \(p\) define \(\text{go}[p]\) = el extremo derecho más lejano entre todas las cámaras que cubren \(p\) (es decir con \(l \le p \le r\)). Se calcula con un prefix-max: en \(\text{arr}[l]\) guardas el máximo \(r\); luego \(\text{go}[p]=\max(\text{go}[p-1],\,\text{arr}[p])\). Si \(\text{go}[p] < p\) para alguna pared, esa pared no la cubre nadie ⇒ impossible.

Pista 4 — Pero hay que probar todos los inicios… ¿no es lento?

Repetir el greedy desde cada inicio \(s\) cuesta \(O(n)\) y son \(n\) inicios ⇒ \(O(n^2)\). ¿Cómo das muchos saltos de golpe? ¿Te suena binary lifting (doblado de saltos)?

Cada "salto" lleva el frente de cobertura de "necesito \(p\)" a "necesito \(\text{go}[p]+1\)". Precomputa \(2^j\) saltos con una tabla dispersa y para cada inicio \(s\) cuenta en \(O(\log n)\) cuántas cámaras hacen falta para llegar a \(s+n-1\). La respuesta es el mínimo sobre todos los \(s\).

Pista final — Solución completa (upsolving)

Idea. Es un cubrimiento mínimo de un círculo con arcos. Desenrollamos el círculo a una recta de longitud \(2n\) y precomputamos, para cada posición, el salto greedy "llega lo más lejos posible". Con binary lifting evaluamos en \(O(\log n)\) el costo de empezar en cada pared \(s\) y tomamos el mínimo.

1) Construcción de los saltos. Para cada cámara \((a,b)\):

  • Forma lineal: \([l,r]=[a,b]\) si \(a\le b\); \([l,r]=[a,\,b+n]\) si \(a>b\).
  • Registra \(\text{arr}[l]=\max(\text{arr}[l],\,r)\) y también la copia \(\text{arr}[l+n]=\max(\text{arr}[l+n],\,\min(r+n,\,2n))\).

Luego un prefix-max sobre \(p=1..2n\) da \(\text{go}[p]\) = extremo derecho máximo alcanzable por una cámara que cubre \(p\). Si \(\text{go}[p]impossible.

2) Por qué el greedy es correcto. Si ya cubriste sin huecos hasta \(p-1\) y la siguiente pared por cubrir es \(p\), elegir la cámara que cubre \(p\) y llega más a la derecha (\(\text{go}[p]\)) nunca empeora: cualquier solución óptima puede intercambiar su cámara que cubre \(p\) por esta sin aumentar el conteo (argumento de intercambio). Solo queda elegir el mejor punto de "costura" \(s\) del círculo, y para eso probamos los \(n\) inicios.

3) Binary lifting. Define \(\text{nxt}[p]=\text{go}[p]+1\) (la siguiente pared a cubrir tras gastar una cámara en \(p\)). Construye \(\text{up}[0][p]=\text{nxt}[p]\) y \(\text{up}[j][p]=\text{up}[j-1][\text{up}[j-1][p]]\). Para un inicio \(s\), el objetivo es alcanzar el frente \(>s+n-1\); cuentas las cámaras bajando por niveles. El mínimo sobre \(s\in[1,n]\) es la respuesta.

Pseudocódigo 
leer n, k
arr[1..2n] = 0                      # extremo derecho máx por posición de inicio
para cada cámara (a, b):
    si a <= b: l, r = a, b
    si no:     l, r = a, b + n      # arco que da la vuelta
    arr[l]   = max(arr[l],   r)
    arr[l+n] = max(arr[l+n], min(r + n, 2n))   # copia desplazada

# prefix-max -> go[p] = extremo derecho máx de una cámara que cubre p
go[0] = 0
para p = 1..2n:
    go[p] = max(go[p-1], arr[p])
    si go[p] < p y p <= n:          # alguna pared sin cobertura
        imprimir "impossible"; terminar
    # (si go[p] < p con p > n, esa posición se completa con la copia)

# saltos: tras gastar 1 cámara en p, el nuevo frente es go[p]+1
LOG = ceil(log2(2n)) + 1
up[0][p] = min(go[p] + 1, 2n+1)    para p = 1..2n
para j = 1..LOG-1:
    para p = 1..2n:
        up[j][p] = up[j-1][ up[j-1][p] ]

mejor = +infinito
para s = 1..n:
    objetivo = s + n              # hay que llegar a frente >= s+n (cubrir s..s+n-1)
    pos = s ; cnt = 0
    para j = LOG-1 .. 0:
        si up[j][pos] < objetivo:
            pos = up[j][pos]
            cnt += 2^j
    cnt += 1                      # un salto más para cruzar el objetivo
    mejor = min(mejor, cnt)

imprimir mejor

Complejidad. Tiempo \(O((n+k)\log n)\); memoria \(O(n\log n)\) por la tabla de saltos (≈ \(2n\) posiciones × \(\sim21\) niveles). Con \(n=10^6\) son del orden de \(40\) millones de enteros: usa int de 32 bits y, si la memoria aprieta, reutiliza el nivel anterior procesando los inicios en cuanto la tabla esté lista.
