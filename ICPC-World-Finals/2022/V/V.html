ICPC World Finals · 2022

Problema V — Three Kinds of Dice

PDF original del concurso: ../problems.pdf

El problema

Cuenta una famosa anécdota que Warren Buffett retó una vez a Bill Gates a un sencillo juego de dados. Buffett tenía tres dados; el primer jugador podía examinarlos y elegir uno de los tres. El segundo jugador escogía después uno de los dados restantes, y ambos lanzaban su dado contra el del otro, buscando sacar el número más alto. Buffett ofreció a Gates elegir primero, pero esto le dio mala espina, así que prefirió ir segundo. Resultó ser una decisión sabia: estos eran dados no transitivos. El primer dado tenía ventaja contra el segundo, el segundo tenía ventaja contra el tercero, ¡pero el primero no tenía ventaja contra el tercero!

Para formalizarlo: definimos un dado como cualquier forma con al menos una cara, tal que cada cara muestra un entero positivo. Al lanzar un dado, se elige una de sus caras de manera uniforme y al azar. Cuando dos dados se enfrentan, el dado cuya cara seleccionada muestra el número más alto gana 1 punto; si los dos números son iguales, cada dado gana \(\tfrac12\) punto.

Para dos dados \(D\) y \(D'\), definimos \(\operatorname{score}(D, D')\) como el número esperado de puntos que gana \(D\) en un solo enfrentamiento contra \(D'\). Si \(\operatorname{score}(D, D') > \tfrac12\), decimos que \(D\) tiene ventaja sobre \(D'\); si \(\operatorname{score}(D, D') = \tfrac12\), los dos dados quedan empatados. Por ejemplo, si \(D\) es el primer dado de la entrada de muestra y \(D'\) es el segundo, entonces \(\operatorname{score}(D, D') = \tfrac49\) y \(\operatorname{score}(D', D) = \tfrac59\), de modo que \(D'\) tiene ventaja sobre \(D\).

Te dan dos dados \(D_1\) y \(D_2\) tales que \(D_1\) tiene ventaja sobre \(D_2\). Quieres un tercer dado \(D_3\) que forme un trío no transitivo con los otros dos. Entre todos los \(D_3\) que tienen ventaja sobre \(D_1\) o empatan con él (es decir, \(\operatorname{score}(D_3, D_1) \ge \tfrac12\)), calcula el menor valor posible de \(\operatorname{score}(D_3, D_2)\). Si ese valor es menor que \(\tfrac12\), ¡puedes armar un trío no transitivo! De forma análoga, entre todos los \(D_3\) tales que \(D_2\) tiene ventaja sobre \(D_3\) o empata con él (es decir, \(\operatorname{score}(D_3, D_2) \le \tfrac12\)), calcula el mayor valor posible de \(\operatorname{score}(D_3, D_1)\).

Entrada

La entrada consta de dos líneas, cada una describiendo un dado. Uno de los dados (el primero o el segundo) tiene ventaja sobre el otro. El dado con la ventaja es \(D_1\) y el otro es \(D_2\) (es decir, tú debes determinar cuál es cuál).

El primer entero de cada línea da \(n\) (\(1 \le n \le 10^5\)), el número de caras del dado. Le siguen \(n\) enteros \(f_i\) (\(1 \le f_i \le 10^9\) para cada \(1 \le i \le n\)), que dan el entero escrito en cada cara.

Salida

Imprime una línea con el menor \(\operatorname{score}(D_3, D_2)\) y el mayor \(\operatorname{score}(D_3, D_1)\) bajo las condiciones anteriores. Los dos valores no tienen por qué usar el mismo dado \(D_3\). Tu respuesta debe tener un error absoluto de a lo más \(10^{-6}\).

Ejemplo 1

Entrada

Salida

Aquí el segundo dado \(\{2,4,9\}\) tiene ventaja sobre el primero \(\{1,1,6,6,8,8\}\) (porque \(\operatorname{score}(\text{primero},\text{segundo})=\tfrac49\)), así que \(D_1=\{2,4,9\}\) y \(D_2=\{1,1,6,6,8,8\}\). El menor \(\operatorname{score}(D_3,D_2)\) con \(\operatorname{score}(D_3,D_1)\ge\tfrac12\) es \(\tfrac{7}{24}=0.2916\overline{6}\); como es \(<\tfrac12\), sí se puede formar un trío no transitivo. El mayor \(\operatorname{score}(D_3,D_1)\) con \(\operatorname{score}(D_3,D_2)\le\tfrac12\) es \(\tfrac34\).

Entrada

3 4 2 3

Salida

Aquí el primer dado \(\{3,5,7,9\}\) tiene ventaja (su \(\operatorname{score}\) contra el segundo es \(\tfrac{10.5}{12}=0.875\)), de modo que \(D_1=\{3,5,7,9\}\) y \(D_2=\{2,3,4\}\). Ambas respuestas valen \(\tfrac12\): no es posible bajar \(\operatorname{score}(D_3,D_2)\) por debajo de \(\tfrac12\) manteniendo \(\operatorname{score}(D_3,D_1)\ge\tfrac12\), así que no se puede armar un trío no transitivo con esta pareja.

Resumen

Te dan dos dados \(D_1\) (con ventaja) y \(D_2\). Buscas un tercer dado \(D_3\) (cualquier multiconjunto de enteros positivos). Pregunta 1: el mínimo de \(\operatorname{score}(D_3,D_2)\) sujeto a \(\operatorname{score}(D_3,D_1)\ge\tfrac12\). Pregunta 2: el máximo de \(\operatorname{score}(D_3,D_1)\) sujeto a \(\operatorname{score}(D_3,D_2)\le\tfrac12\). Las dos respuestas pueden usar dados \(D_3\) distintos.

Lo que importa de los límites \(n \le 10^5\) y valores hasta \(10^9\) ⇒ ordenar y trabajar por valores distintos \(O(n\log n)\). Como \(D_3\) es libre, no hay que "buscar" un dado: hay que caracterizar TODO lo alcanzable.

Pistas

Pista 1 — ¿Qué tan libre es \(D_3\)?

¿Ya notaste que \(D_3\) puede tener tantas caras como quieras, con cualquier valor? ¿Qué pasa si un dado se ve como una distribución de probabilidad sobre los valores de sus caras?

Un dado de \(m\) caras es exactamente una distribución uniforme sobre esas \(m\) caras; y como \(m\) es libre, puedes lograr (en el límite) cualquier distribución de probabilidad con pesos racionales sobre los enteros positivos. Es decir, \(D_3\) no es "un objeto rígido": es una mezcla a tu gusto de caras individuales.

Pista 2 — Linealidad: el par \((x,y)\)

Para una sola cara con valor \(v\), define \(a(v)=\operatorname{score}(v,D_1)\) y \(b(v)=\operatorname{score}(v,D_2)\). Si \(D_3\) usa la cara \(v\) con probabilidad \(p_v\), ¿cuánto vale \(\operatorname{score}(D_3,D_1)\)?

Por linealidad de la esperanza, \(\operatorname{score}(D_3,D_1)=\sum_v p_v\,a(v)\) y \(\operatorname{score}(D_3,D_2)=\sum_v p_v\,b(v)\). O sea, el par \(\big(\operatorname{score}(D_3,D_1),\operatorname{score}(D_3,D_2)\big)\) es una combinación convexa de los puntos \(\big(a(v),b(v)\big)\).

Pista 3 — El conjunto alcanzable es un polígono

Si todo par alcanzable es una combinación convexa de los puntos \((a(v),b(v))\), ¿qué figura geométrica forma el conjunto de TODO lo alcanzable?

Es la envolvente convexa de esos puntos. Ahora bien, ¿cuántos puntos \((a(v),b(v))\) distintos hay realmente? \(a\) y \(b\) son funciones escalera que solo cambian en los valores de cara de \(D_1\) y \(D_2\). Así que basta probar \(O(n)\) valores de \(v\): cada valor distinto \(w\) (para contar los empates con \(\tfrac12\)) y un valor "justo arriba" \(w{+}1\) (para contar como estrictamente mayor), más quizá un valor por debajo del mínimo que da el punto \((0,0)\).

Pista 4 — Convertir las preguntas en cortes

Ya tienes el polígono convexo de pares \((x,y)=(\operatorname{score}(D_3,D_1), \operatorname{score}(D_3,D_2))\). La pregunta 1 pide minimizar \(y\) con \(x\ge\tfrac12\), y la 2 maximizar \(x\) con \(y\le\tfrac12\). ¿Dónde se alcanza el óptimo de una función lineal sobre un convexo recortado por una recta?

En un vértice del polígono ya recortado: o un vértice original que cumple la restricción, o un punto donde la recta del corte (\(x=\tfrac12\) o \(y=\tfrac12\)) cruza una arista de la envolvente. Evalúa esos candidatos y listo. Nota que ambos cortes siempre son factibles: \(D_3=D_1\) da \(x=\tfrac12\) y \(D_3=D_2\) da \(y=\tfrac12\).

Por qué es correcto. Cualquier \(D_3\) asigna probabilidad \(p_v\) a cada valor \(v\); por linealidad de la esperanza el par resultante es \(\sum_v p_v\,(a(v),b(v))\), una combinación convexa de puntos generadores. Recíprocamente, cualquier combinación convexa con pesos racionales se realiza con un dado concreto (y los irracionales por límite; el problema permite error \(10^{-6}\)). Por tanto el conjunto alcanzable es exactamente la envolvente convexa \(H\) de los generadores.

  • Pregunta 1: \(\min y\) sobre \(H\cap\{x\ge\tfrac12\}\). Es el valor de la cadena inferior (lower hull) en \(x=\tfrac12\), salvo que el vértice de \(y\) mínimo global ya tenga \(x\ge\tfrac12\).
  • Pregunta 2: \(\max x\) sobre \(H\cap\{y\le\tfrac12\}\). Es el borde derecho de \(H\) evaluado en \(y=\tfrac12\), salvo que el vértice de \(x\) máximo global ya tenga \(y\le\tfrac12\).

El óptimo de una función lineal sobre un convexo cortado por un semiplano se alcanza en un vértice del recorte: o un vértice original que satisface la restricción, o la intersección de la recta de corte con una arista de \(H\). Basta evaluar esos candidatos.

Verificación con el Ejemplo 1. \(D_1=\{2,4,9\}\), \(D_2=\{1,1,6,6,8,8\}\). Los generadores incluyen \((a,b)=(0,\tfrac16)\) en \(v=1\) y \((a,b)=(\tfrac23,\tfrac13)\) en \(v=5\). La cadena inferior pasa por esos dos puntos; su valor en \(x=\tfrac12\) es \(\tfrac16+\tfrac14\cdot\tfrac12=\tfrac{7}{24}=0.2916\overline6\) (respuesta 1). Para la respuesta 2, el segmento entre \((\tfrac23,\tfrac13)\) y \((1,1)\) cruza \(y=\tfrac12\) en \(x=\tfrac34\). Coincide con la salida esperada.

Pseudocódigo 
leer D1, D2            # si score(D1,D2) < 1/2, intercambiar para que D1 tenga ventaja
ordenar caras de D1 y de D2

func a(v):  # = score(cara v, D1)
    menores = #caras de D1 < v
    iguales = #caras de D1 = v        # con búsqueda binaria
    regresa (menores + iguales/2) / |D1|
func b(v):  análogo con D2

# 1) generar puntos candidatos
V = conjunto de valores distintos en D1 ∪ D2
cand = {}
si min(V) > 1:  cand += { (0, 0) }            # cara por debajo de todo
para cada w en V:
    cand += { ( a(w),   b(w)   ) }            # empate exacto en w
    cand += { ( a(w+1), b(w+1) ) }            # estrictamente mayor que w
P = lista de puntos distintos de cand

# 2) envolvente convexa
H = envolvente_convexa(P)                     # polígono (Andrew monotone chain)

# 3) pregunta 1: min y con x >= 1/2
resp1 = +infinito
para cada vértice u de H con u.x >= 1/2:
    resp1 = min(resp1, u.y)
para cada arista (u,w) de H que cruza x = 1/2:
    t = (1/2 - u.x) / (w.x - u.x)
    resp1 = min(resp1, u.y + t*(w.y - u.y))

# 4) pregunta 2: max x con y <= 1/2  (simétrico)
resp2 = -infinito
para cada vértice u de H con u.y <= 1/2:
    resp2 = max(resp2, u.x)
para cada arista (u,w) de H que cruza y = 1/2:
    t = (1/2 - u.y) / (w.y - u.y)
    resp2 = max(resp2, u.x + t*(w.x - u.x))

imprimir resp1, resp2

Complejidad: \(O(n\log n)\) en tiempo (ordenar + búsquedas binarias + envolvente) y \(O(n)\) en memoria.
