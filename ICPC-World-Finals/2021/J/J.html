ICPC World Finals · 2021

Problema J — Splitstream

PDF original del concurso: ../problems.pdf

El problema

Un sistema splitstream es una red acíclica de nodos que procesa secuencias finitas de números. Hay dos tipos de nodos (ilustrados en la Figura J.1):

  • Nodo de división (split): toma una secuencia de números como entrada y la reparte alternando entre sus dos salidas. El primer número va a la salida 1, el segundo a la salida 2, el tercero a la salida 1, el cuarto a la salida 2, y así sucesivamente, en ese orden.
  • Nodo de fusión (merge): toma dos secuencias de números como entrada y las intercala para formar su única salida. La salida contiene el primer número de la entrada 1, luego el primero de la entrada 2, luego el segundo de la entrada 1, luego el segundo de la entrada 2, y así sucesivamente. Si una de las secuencias de entrada es más corta que la otra, entonces, una vez agotada la secuencia corta, los números restantes de la secuencia más larga simplemente se transmiten sin intercalarse.

La red completa tiene una sola entrada global, que es la secuencia de enteros positivos \(1, 2, 3, \dots, m\). Se puede consultar cualquier salida de cualquier nodo. Una consulta busca identificar el \(k\)-ésimo número en la secuencia de una salida dada y un \(k\) dado. Tu tarea es responder esas consultas de forma eficiente.

Entrada

La primera línea contiene tres enteros \(m\), \(n\) y \(q\), donde \(m\) (\(1 \le m \le 10^9\)) es la longitud de la secuencia de entrada, \(n\) (\(1 \le n \le 10^4\)) es el número de nodos y \(q\) (\(1 \le q \le 10^3\)) es el número de consultas.

Las siguientes \(n\) líneas describen la red, un nodo por línea. Un nodo de división tiene el formato S x y z, donde \(x\), \(y\) y \(z\) identifican, respectivamente, su entrada, su primera salida y su segunda salida. Un nodo de fusión tiene el formato M x y z, donde \(x\), \(y\) y \(z\) identifican, respectivamente, su primera entrada, su segunda entrada y su salida. Los identificadores \(x\), \(y\) y \(z\) son enteros positivos distintos. La entrada global se identifica con \(1\), y los demás identificadores de entrada/salida forman una secuencia consecutiva que empieza en \(2\). Todo identificador de entrada excepto el \(1\) aparece como salida de exactamente un nodo. Todo identificador de salida aparece como entrada de a lo sumo un nodo.

Cada una de las siguientes \(q\) líneas describe una consulta. Cada consulta consiste en dos enteros \(x\) y \(k\), donde \(x\) (\(2 \le x \le 10^5\)) es un identificador de salida válido y \(k\) (\(1 \le k \le 10^9\)) es el índice del número deseado en esa secuencia. La indexación de una secuencia empieza en \(1\).

Salida

Para cada consulta \(x\), \(k\), imprime una línea con el \(k\)-ésimo número de la secuencia de salida identificada por \(x\), o bien none si no existe ningún elemento con ese índice.

Resumen

Un DAG de nodos split (reparten 1 entrada en 2 salidas alternando) y merge (intercalan 2 entradas en 1 salida). La entrada global es \(1,2,\dots,m\). Para cada consulta \((x,k)\) hay que devolver el valor en la posición \(k\) de la secuencia que circula por el identificador \(x\), o none si esa secuencia tiene menos de \(k\) elementos.

Lo que importa de los límites \(m\) hasta \(10^9\) ⇒ las secuencias son enormes, jamás las materialices. \(n \le 10^4\), \(q \le 10^3\): se vale recorrer una ruta por consulta (\(O(n)\) por consulta).

Pistas

Pista 1 — ¿Construir las secuencias o no?

Con \(m\) hasta \(10^9\), ¿de verdad puedes generar y guardar cada secuencia? ¿Qué pasa si en vez de calcular toda la secuencia solo rastreas una posición concreta?

Materializar es imposible. Pero una consulta solo pide el \(k\)-ésimo elemento de una salida: te basta seguir hacia atrás de dónde vino ese único elemento.

Pista 2 — ¿Qué necesitas saber de cada arista?

Para un nodo merge, ¿puedes decir de cuál entrada sale el \(k\)-ésimo elemento sin conocer las longitudes de sus dos entradas? ¿Y cómo calculas esas longitudes sin construir nada?

Cada identificador (arista) lleva una secuencia de cierta longitud \(L\). Un split parte \(L\) en \(\lceil L/2\rceil\) (salida 1) y \(\lfloor L/2\rfloor\) (salida 2). Un merge suma: \(L_{\text{out}} = L_a + L_b\). La entrada global tiene longitud \(m\). Las longitudes se propagan por el DAG con memoización.

Pista 3 — Invertir un split y un merge

Si quieres el elemento \(k\) de la primera salida de un split, ¿qué posición ocupaba en la entrada del split? ¿Y de la segunda salida? Para el merge, ¿qué pasa cuando \(k\) cae en la zona ya intercalada y qué pasa cuando una entrada ya se agotó?

  • Split, salida 1: el \(k\)-ésimo de la salida es el \((2k-1)\)-ésimo de la entrada (índices impares).
  • Split, salida 2: es el \((2k)\)-ésimo de la entrada (índices pares).
  • Merge: sea \(t=\min(L_a,L_b)\). Los primeros \(2t\) elementos están intercalados; si \(k\le 2t\): \(k\) impar viene de la entrada \(a\) en posición \((k+1)/2\); \(k\) par viene de \(b\) en \(k/2\). Si \(k>2t\), el resto es la cola de la entrada más larga.
Pista 4 — ¿Dónde termina el rastreo?

Si sigues retrocediendo posición tras posición, ¿a qué arista llegas siempre al final? ¿Cuánto vale el elemento \(k\) de la entrada global \(1,2,\dots,m\)?

El rastreo termina en el identificador \(1\): ahí la secuencia es \(1,2,\dots,m\), así que el elemento en la posición \(k\) vale exactamente \(k\). Antes de empezar, si \(k\) supera la longitud de \(x\), la respuesta es none.

Pista final — Solución completa (upsolving)

Idea. Como cada identificador (excepto el \(1\)) es la salida de exactamente un nodo, cada arista tiene un único productor. Eso convierte el rastreo hacia atrás en una ruta sin ambigüedad por el DAG.

Paso 1 — Longitudes. Define \(\text{len}(\text{id})\): \(\text{len}(1)=m\); si la arista la produce un split desde la entrada \(a\), la salida 1 mide \(\lceil \text{len}(a)/2\rceil\) y la salida 2 mide \(\lfloor \text{len}(a)/2\rfloor\); si la produce un merge de \(a\) y \(b\), mide \(\text{len}(a)+\text{len}(b)\). Se calcula con memoización (recursión con pila explícita o un orden topológico) en \(O(n)\). Usa enteros de 64 bits: las longitudes pueden crecer al sumar.

Paso 2 — Responder una consulta \((x,k)\). Si \(k>\text{len}(x)\), imprime none. Si no, repite hasta llegar al identificador \(1\): busca el productor de la arista actual e invierte la operación, transformando \((\text{id},k)\) en la pareja \((\text{id}',k')\) de su origen, según las reglas de la Pista 3. Al llegar al identificador \(1\), la respuesta es el \(k\) actual.

Por qué es correcto. Las reglas de inversión son la definición exacta de cómo split y merge colocan cada elemento, así que cada paso preserva «qué número del flujo original estamos siguiendo». El índice \(k\) nunca se sale del rango: si entra dentro de \(\text{len}(x)\), tras cada inversión sigue dentro de la longitud de la arista origen (por construcción de \(\lceil\cdot\rceil/\lfloor\cdot\rfloor\) y de \(L_a+L_b\)). Como el grafo es acíclico, el rastreo siempre avanza hacia el identificador \(1\) y termina.

Pseudocódigo 
// prod[id] = (tipo, a, b, c) del nodo que PRODUCE la arista id
//   split  S a b c : a=entrada, b=salida1, c=salida2
//   merge  M a b c : a=entrada1, b=entrada2, c=salida

func len(id):
    si id == 1: regresa m
    si memo[id] definido: regresa memo[id]
    (tipo, a, b, c) = prod[id]
    si tipo == SPLIT:
        L = len(a)
        si id == b: r = ceil(L/2)      // (L+1)/2
        else:       r = floor(L/2)     //  L/2
    else: // MERGE, id == c
        r = len(a) + len(b)
    memo[id] = r; regresa r

func consulta(x, k):
    si k > len(x): regresa "none"
    id = x
    mientras id != 1:
        (tipo, a, b, c) = prod[id]
        si tipo == SPLIT:
            si id == b: k = 2*k - 1   // salida 1 -> índices impares
            else:       k = 2*k       // salida 2 -> índices pares
            id = a
        else: // MERGE de a y b -> c
            La = len(a); Lb = len(b); t = min(La, Lb)
            si k <= 2*t:
                si k impar: id = a; k = (k+1)/2
                else:       id = b; k = k/2
            else:
                si La > Lb: id = a       // cola de la entrada más larga
                else:       id = b
                k = k - t
    regresa k   // en el id 1 la secuencia es 1..m, valor = k

// nota: len() conviene calcularla iterativa para evitar
// desbordar la pila cuando la profundidad del DAG llega a ~n.

Complejidad. Longitudes en \(O(n)\) (cada arista se evalúa una vez). Cada consulta recorre una ruta del DAG, a lo sumo \(O(n)\) pasos, así que \(O(n\cdot q)\) en total (\(\le 10^4 \cdot 10^3 = 10^7\)). Memoria \(O(\text{máx id})\), del orden de \(O(n)\).
