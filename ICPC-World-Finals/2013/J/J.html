Problema J — Pollution Solution

PDF original del concurso: ../problems.pdf

El problema

Como empleado de Aqueous Contaminate Management, tu trabajo es vigilar la contaminación que se vierte (a veces por accidente, a veces a propósito) en ríos, lagos y océanos. Una de tus tareas es medir el impacto de esa contaminación sobre distintos ecosistemas acuáticos: arrecifes de coral, zonas de desove, etcétera.

El modelo que usas en tu análisis se ilustra en la Figura J.1. La línea de costa (la línea horizontal de la figura) se encuentra sobre el eje \(x\), y la fuente de la contaminación está en el origen \((0,0)\). La dispersión de la contaminación hacia el agua se representa con un semicírculo, y el polígono representa el ecosistema que nos importa. Debes determinar el área del ecosistema que queda contaminada, que es la región azul oscuro de la figura.

Entrada

La entrada consiste en un solo caso de prueba. El caso comienza con una línea que contiene dos enteros \(n\) y \(r\), donde \(3 \le n \le 100\) es el número de vértices del polígono y \(1 \le r \le 1000\) es el radio del campo de contaminación. A continuación vienen \(n\) líneas, cada una con dos enteros \(x_i, y_i\), que dan las coordenadas de los vértices del polígono en sentido antihorario, donde \(-1500 \le x_i \le 1500\) y \(0 \le y_i \le 1500\). El polígono no se autointersecta ni se toca a sí mismo. Ningún vértice queda exactamente sobre la frontera del círculo.

Salida

Muestra el área del polígono que cae dentro del semicírculo centrado en el origen con radio \(r\). Da el resultado con un error absoluto de a lo más \(10^{-3}\).

Resumen

Te dan un polígono simple de \(n\) vértices (en orden antihorario, todo con \(y \ge 0\)) y un círculo de radio \(r\) centrado en el origen. Hay que calcular el área de la intersección entre el polígono y el círculo, con error \(\le 10^{-3}\).

Lo que importa de los límites \(n \le 100\): casi cualquier cosa pasa en tiempo. El reto es puramente de precisión geométrica, no de velocidad. Usa double y cuida los casos de borde (segmento que cruza, toca o no toca el círculo).

Pistas

Pista 1 — Semicírculo vs. círculo completo

¿Ya notaste que todos los vértices cumplen \(0 \le y_i\)? ¿Qué significa eso sobre dónde vive el polígono completo?

Como cada arista une dos puntos con \(y \ge 0\), el polígono entero está en el semiplano superior \(y \ge 0\). Entonces intersectarlo con el semicírculo superior es exactamente lo mismo que intersectarlo con el círculo completo. Olvídate del "semi": el problema es "área de intersección entre un polígono y un disco".

Pista 2 — Descomponer en triángulos desde el centro

El área de un polígono se puede escribir como una suma con signo de triángulos \((O, P_i, P_{i+1})\) usando el producto cruz. ¿Y si haces lo mismo pero recortando cada triángulo al disco?

El área del polígono es \(\frac12\left|\sum_i P_i \times P_{i+1}\right|\), es decir, una suma con signo de triángulos con vértice en el origen \(O\). Si para cada arista calculas el área (con signo) de la parte de ese triángulo \((O, P_i, P_{i+1})\) que cae dentro del disco, y las sumas, obtienes el área de la intersección polígono–disco. El signo de cada término se cancela igual que en la fórmula del zapatero, así que la suma da el resultado correcto.

Pista 3 — El área "triángulo ∩ disco" por casos

Para un solo triángulo \((O,A,B)\) con \(O\) en el centro del círculo, ¿cuántas situaciones hay según si \(A\) y \(B\) están dentro o fuera, y si el segmento \(AB\) cruza el círculo?

Necesitas una función \(f(A,B)\) que devuelva el área con signo del sector/triángulo \(OAB\) recortado al disco. Hay 4 casos según \(A,B\) dentro/fuera, y cuando ambos están fuera el segmento puede cortar el círculo en 0 o 2 puntos. La pieza clave es que, cuando una parte cae fuera del disco, esa porción se sustituye por un sector circular de área \(\frac12 r^2\theta\) (con \(\theta\) el ángulo barrido), no por el triángulo.

Pista final — Solución completa (upsolving)

Idea. Reducimos a "área de intersección de un polígono con un disco de radio \(r\) centrado en el origen" (válido porque el polígono está en \(y\ge 0\)). Recorremos las aristas \(P_i \to P_{i+1}\) y sumamos \(f(P_i, P_{i+1})\), el área con signo del wedge \(O\,P_i\,P_{i+1}\) recortada al disco. La respuesta es el valor absoluto de la suma.

La función \(f(A,B)\). Sea \(\operatorname{cross}(A,B)=A_xB_y-A_yB_x\) (el doble del área con signo del triángulo \(OAB\)). Definimos un sector con signo entre dos vectores \(u,v\) del disco: \(\operatorname{sec}(u,v)=\dfrac{r^2}{2}\,\theta\cdot\operatorname{signo}(\operatorname{cross}(u,v))\), donde \(\theta=\angle(u,v)\in[0,\pi]\) es el ángulo entre ellos.

Para una arista, sean \(A,B\) sus extremos. Marcamos si están dentro (\(|A|

  • Ambos dentro: todo el triángulo \(OAB\) está en el disco \(\Rightarrow f=\tfrac12\operatorname{cross}(A,B)\).
  • \(A\) dentro, \(B\) fuera: el segmento sale por un punto \(P\) (intersección segmento–círculo). La parte \(OAP\) es triángulo y \(OPB\) se vuelve sector \(\Rightarrow f=\tfrac12\operatorname{cross}(A,P)+\operatorname{sec}(P,B)\).
  • \(A\) fuera, \(B\) dentro: simétrico, entra por \(P\) \(\Rightarrow f=\operatorname{sec}(A,P)+\tfrac12\operatorname{cross}(P,B)\).
  • Ambos fuera: si el segmento corta el círculo en dos puntos \(P,Q\) (con \(P\) más cerca de \(A\)), entonces \(f=\operatorname{sec}(A,P)+\tfrac12\operatorname{cross}(P,Q)+\operatorname{sec}(Q,B)\). Si no lo corta (o lo toca en un punto), todo es arco \(\Rightarrow f=\operatorname{sec}(A,B)\).

Cortes segmento–círculo. Parametriza \(X(t)=A+t(B-A)\), resuelve \(|X(t)|^2=r^2\) (cuadrática en \(t\)) y quédate con las raíces en \([0,1]\), ordenadas por \(t\). Esas son \(P\) (y \(Q\)).

Por qué es correcto. Cada arista contribuye el área con signo de la región barrida desde \(O\) hasta el tramo de arista, recortada al disco. Al sumar sobre el polígono cerrado, las contribuciones fuera del polígono se cancelan (igual que en la fórmula del zapatero) y queda exactamente el área de polígono \(\cap\) disco. El sentido antihorario hace que la suma salga positiva; tomar \(|\cdot|\) la deja robusta.

Pseudocódigo 
leer n, r
leer P[0..n-1]                      # vértices, antihorario

func cross(A,B):  return A.x*B.y - A.y*B.x
func dentro(A):   return A.x*A.x + A.y*A.y < r*r      # vértice nunca está justo en el borde

func sector(u,v):                  # área con signo del sector entre u y v
    ang = atan2(cross(u,v), dot(u,v))   # en (-pi, pi], ya trae el signo
    return 0.5 * r*r * ang

func cortes(A,B):                  # puntos de A->B sobre el círculo, t en [0,1], ordenados
    d = B - A
    a = dot(d,d); b = 2*dot(A,d); c = dot(A,A) - r*r
    disc = b*b - 4*a*c
    si disc <= 0: return []        # no cruza (o tangente: área despreciable)
    s = sqrt(disc)
    res = []
    para t en { (-b - s)/(2a), (-b + s)/(2a) }:
        si 0 <= t <= 1: res.add(A + t*d)
    return res                     # ya viene ordenado por t

func f(A,B):
    inA = dentro(A);  inB = dentro(B)
    si inA y inB:
        return 0.5*cross(A,B)
    si inA y no inB:
        P = cortes(A,B)[0]
        return 0.5*cross(A,P) + sector(P,B)
    si no inA y inB:
        P = cortes(A,B)[ultimo]
        return sector(A,P) + 0.5*cross(P,B)
    si no:                          # ambos fuera
        ps = cortes(A,B)
        si ps tiene 2 puntos:       # P=ps[0] (cerca de A), Q=ps[1]
            return sector(A,ps[0]) + 0.5*cross(ps[0],ps[1]) + sector(ps[1],B)
        si no:
            return sector(A,B)

total = 0
para i en 0..n-1:
    total += f(P[i], P[(i+1) mod n])
imprimir abs(total)

Complejidad: \(O(n)\) tiempo y \(O(n)\) memoria. Con \(n \le 100\) es instantáneo; lo único delicado es la precisión, que double cubre de sobra para el error \(10^{-3}\) pedido.
