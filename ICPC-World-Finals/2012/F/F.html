ICPC World Finals · 2012

Problema F — Keys

PDF original del concurso: ../problems.pdf

El problema

Adam tiene un montón de llaves enganchadas a llaveros (anillos), y algunos de esos anillos pueden estar conectados entre sí. Los anillos son llaveros comunes en espiral, así que una llave se puede enganchar o desenganchar de un anillo deslizándola por la espiral. De la misma forma, dos anillos se pueden conectar o desconectar entre sí.

Adam quiere regalarle algunas llaves a Brenda. Como manipular las llaves y los anillos es molesto (y además peligroso para las uñas), Adam busca la manera de minimizar la cantidad de operaciones sobre llaves y anillos.

Cada vez que se engancha una llave, se desengancha una llave, se conectan dos anillos o se desconectan dos anillos, eso cuenta como una operación. Como manipular dos anillos es mucho más fácil que deslizar una llave, primero queremos minimizar la cantidad de llaves que se desenganchan y enganchan. De entre todas las soluciones que tengan esa misma cantidad mínima de operaciones sobre llaves, hay que encontrar la que use la cantidad mínima de conexiones y desconexiones de anillos.

Cuando terminen todas las operaciones, Adam y Brenda deben quedar cada uno con un único grupo conectado de anillos y llaves. La única excepción es cuando alguno de los dos se queda sin ninguna llave: en ese caso no necesita ningún anillo. Cada llave debe quedar enganchada a exactamente un anillo. Algunos anillos (pero no llaves) se pueden considerar sobrantes y quedar desconectados de los dos grupos.

El lado izquierdo de la siguiente figura muestra una configuración inicial de cuatro llaves sobre tres anillos. Adam quiere darle a Brenda las dos llaves etiquetadas N y R. Esto se puede lograr con dos operaciones de llave y una operación de anillo, dando como resultado la configuración que se ve del lado derecho de la figura.

Entrada

Cada caso de prueba contiene una o más líneas; cada línea es una cadena de dos letras. Las letras minúsculas (a–z) representan anillos y las mayúsculas (A–Z) representan llaves. Las dos letras de una línea indican una de dos cosas: o bien una llave enganchada a un anillo (una mayúscula y una minúscula), o bien dos anillos conectados entre sí (dos minúsculas). El final de cada caso de prueba se marca con una línea que contiene el dígito cero.

Las llaves con letras de la A a la M se quedan con Adam, y las llaves con letras de la N a la Z se le dan a Brenda.

Ninguna línea contiene dos letras mayúsculas. Ningún par de letras se especifica más de una vez dentro del mismo caso de prueba. Cada llave está conectada a exactamente un anillo. No hay "ciclos" en la configuración de anillos (desconectar cualesquiera dos anillos siempre aumenta la cantidad de grupos conectados). Todas las llaves y anillos que existen se mencionan al menos una vez.

Salida

Para cada caso de prueba, muestra el número de caso seguido de la cantidad mínima de operaciones de enganchar/desenganchar llaves y la cantidad mínima de operaciones de conectar/desconectar anillos.

Si no hay manera de repartir las llaves como se pide, muestra el número de caso y la palabra impossible en lugar de los dos enteros.

Etiquetas Grafos Árboles Greedy DP Parsing

Resumen

Te dan un bosque cuyos nodos son anillos (minúsculas) conectados entre sí, y llaves (mayúsculas) colgadas cada una de un anillo. Las llaves \(A\)–\(M\) son de Adam y \(N\)–\(Z\) de Brenda. Hay que repartir todo en un grupo conectado para Adam (con todas sus llaves) y otro para Brenda (con todas las suyas), minimizando primero las operaciones de llave (desenganchar/enganchar) y, en segundo lugar, las de anillo (desconectar/conectar). Reporta ambos mínimos o impossible.

Lo que importa de los límites A lo más 26 anillos y 26 llaves: el tamaño es minúsculo, así que la dificultad es modelar, no el rendimiento. Todo es \(O(\text{anillos}+\text{llaves})\).

Pistas

Pista 1 — ¿Qué cuesta mover una llave?

¿Ya pensaste qué pasa con una llave que ya está en un anillo que terminará en el grupo de su dueño? ¿Y cuánto cuesta reubicar una que no?

Una llave que se queda en su anillo cuesta 0 operaciones de llave. Reubicarla cuesta exactamente 2 (un desenganche y un enganche). Como la prioridad #1 es minimizar operaciones de llave, conviene pensar primero en qué llaves no hay que mover.

Pista 2 — Cada anillo "elige un color"

Al final, ¿a qué grupo puede pertenecer un anillo? ¿Puede un mismo anillo conservar llaves de Adam y de Brenda a la vez? Y si ambos dueños tienen llaves, ¿basta con que cada anillo elija su color más barato?

No: cada anillo termina en exactamente un grupo (Adam, Brenda o sobrante). Si un anillo tiene \(a\) llaves de Adam y \(b\) de Brenda, conserva las del color que elija y desengancha+engancha el resto. Las operaciones de llave de ese anillo son \(2\cdot(\text{llaves del color descartado})\). El piso (baseline) es que cada anillo elija su color más barato: \(\;2\sum_{\text{anillo } r}\min(a_r,b_r)\).

Pero la decisión NO es del todo independiente. Hay una restricción global: si ambos dueños tienen llaves, la coloración final debe incluir al menos un anillo etiquetado \(A\) y al menos uno \(B\) (cada dueño con llaves necesita un grupo no vacío). Si todos los anillos con llaves prefieren estrictamente el mismo color y no hay anillos sin llaves para regalar, el baseline es infactible y hay que pagar una penalización para darle un anillo al dueño que quedó sin ninguno (ver la pista final).

Pista 3 — ¿Cuándo es imposible?

Si ambos dueños tienen llaves, ¿cuántos anillos necesitas como mínimo? ¿Puedes crear anillos nuevos?

No se pueden crear anillos: solo reacomodar los que hay. Si Adam y Brenda tienen llaves, necesitas dos grupos conectados separados, o sea al menos 2 anillos. Es impossible exactamente cuando ambos tienen llaves y solo existe 1 anillo. En cualquier otro caso siempre se puede.

Pista 4 — Las operaciones de anillo, como una partición coloreada

Fijado qué color conserva cada anillo, ¿qué aristas anillo-anillo tienes que cortar y cuántas tienes que crear? ¿Una arista entre dos anillos del mismo grupo, te cuesta algo dejarla?

Etiqueta cada anillo con \(A\), \(B\) o \(L\) (sobrante). Las aristas anillo-anillo del bosque inicial: las que unen dos anillos del mismo color se conservan gratis; las que cruzan colores hay que cortarlas (1 operación cada una). Además, todos los anillos \(A\) deben quedar en un solo componente y todos los \(B\) en otro, así que por cada componente de más hay que añadir una arista. La pregunta es cómo colorear los anillos libres (sin llaves) y los empatados (igual número de cada color) para minimizar ese costo.

Pista final — Solución completa (upsolving)

Modelo. Construye el grafo de anillos (solo aristas minúscula-minúscula); por las garantías del problema es un bosque. Cada llave cuelga de un anillo y tiene dueño: \(A\)–\(M\) Adam, \(N\)–\(Z\) Brenda. Sea \(R\) el número de anillos y \(M\) el número de aristas anillo-anillo.

Imposible. Sea \(\text{hasA}\) = "Adam tiene alguna llave" y \(\text{hasB}\) análogo. Si \(\text{hasA}\) y \(\text{hasB}\) y \(R<2\), imprime impossible. (Hacen falta dos grupos separados y no se pueden crear anillos.)

Operaciones de llave (objetivo primario). El piso (baseline) es que cada anillo elija su color más barato: por cada anillo \(r\) con \(a_r\) llaves de Adam y \(b_r\) de Brenda se mueven \(\min(a_r,b_r)\) llaves, y los anillos sin llaves quedan \(L\): \[\text{baseline}=2\sum_{r}\min(a_r,b_r).\] Cada anillo con mayoría estricta conserva su color mayoritario (forzado); si \(a_r=b_r>0\) hay empate (puede ser \(A\) o \(B\), no cambia el costo de llaves); si no tiene llaves, es libre (\(A\), \(B\) o \(L\)).

Cuidado: el baseline puede ser infactible. Si ambos dueños tienen llaves, la coloración final debe contener al menos un anillo \(A\) y al menos uno \(B\). Tras el baseline, revisa la restricción global: para cada dueño con llaves cuyo color no aparezca en ningún anillo, suma la penalización mínima de darle uno:

  • \(0\) si existe algún anillo sin llaves (se le asigna gratis a ese color) —o si existe un anillo empatado, que también se asigna gratis—;
  • en caso contrario, \(\displaystyle\min_{r\ \text{con llaves}} 2\,|a_r-b_r|\) (flipear el anillo de cambio más barato, soltando su mayoría en vez de la minoría; los empates dan \(0\)).

Entonces \(\text{ops\_llave}=\text{baseline}+\text{penalización}\). Importante: el anillo que se flipea por esta restricción queda con su color fijado (\(\text{perm}[r]=\{A\}\) o \(\{B\}\)) antes de correr el DP de anillos, para que el objetivo secundario use exactamente esa coloración factible. La fórmula simple \(2\sum_r\min(a_r,b_r)\) subcuenta en estos casos.

Contraejemplo (entrada aA / aN / aO / bB / bP / bQ / 0): el anillo \(a=\{A,N,O\}\) y el anillo \(b=\{B,P,Q\}\) tienen ambos mayoría \(B\). El baseline daría \(2(\min(1,2)+\min(1,2))=4\), pero forzar ambos a \(B\) deja a Adam sin ningún anillo: infactible. Hay que flipear un anillo a \(A\) (soltando sus \(2\) llaves de Brenda en vez de \(1\) de Adam), penalización \(2\,|1-2|=2\), total \(\text{ops\_llave}=6\).

Operaciones de anillo (objetivo secundario). Fijada una coloración válida, sean \(AA,BB,LL\) las aristas monocromáticas conservadas, \(n_A,n_B,n_L\) la cantidad de anillos de cada color. Como el bosque no tiene ciclos, las aristas \(A\)-\(A\) forman a su vez un bosque, así que el número de componentes de color \(A\) es \(n_A-AA\) (igual para \(B\)). Entonces:

  • Aristas a cortar = \(M-(AA+BB+LL)\) (todo lo no monocromático).
  • Aristas a añadir para unir el grupo de Adam = \(n_A-AA-1\) (si Adam tiene llaves), y \(n_B-BB-1\) para Brenda.

Sumando y usando \(n_A+n_B+n_L=R\), el costo de anillos se reduce a una forma muy limpia: \[\text{ops\_anillo}=(M+R-K)\;-\;f,\qquad f=2AA+2BB+LL+n_L,\] donde \(K\) = cuántos dueños tienen al menos una llave (1 o 2). \(M\), \(R\) y \(K\) son constantes, así que basta maximizar \(f\).

Por qué \(f\) decompone bien. Cada anillo \(L\) aporta \(+1\) (por \(n_L\)); cada arista monocromática aporta \(+2\) si es \(A\)/\(B\) y \(+1\) si es \(L\)/\(L\). Todo es por nodo y por arista, sin acoplamiento global, así que un DP de árbol lo maximiza de forma exacta. Intuición de \(f\): un anillo sin llaves aislado conviene dejarlo sobrante (\(+1\), evita una unión); pero si tiene un vecino del color de un grupo, conviene unirlo a ese grupo (\(+2\) por la arista que se aprovecha en vez de cortarse).

DP de árbol. Enraíza cada componente. Para cada nodo y cada etiqueta permitida, combina con los hijos sumando el bono de arista. Las etiquetas permitidas: un anillo solo puede ser \(A\) si Adam tiene llaves, y \(B\) si Brenda las tiene (si no, esa coloración inflaría \(f\) sin pagar el \(-1\) correspondiente); los anillos con mayoría estricta quedan forzados, y el anillo flipeado por la restricción global del objetivo primario también queda con su color fijado en \(\text{perm}[\cdot]\). Así el DP solo explora coloraciones factibles (al menos un anillo \(A\) si \(\text{hasA}\) y al menos uno \(B\) si \(\text{hasB}\)).

Pseudocódigo 
leer caso -> anillos, aristas_anillo (bosque), llaves(dueño, anillo)
hasA = existe llave de Adam ;  hasB = existe llave de Brenda
R = #anillos ;  M = #aristas_anillo
si hasA y hasB y R < 2:  imprime "impossible" ;  continuar

# --- objetivo 1: operaciones de llave ---
para cada anillo r:  a_r, b_r = #llaves Adam/Brenda en r
baseline = 2 * sum_r min(a_r, b_r)               # cada anillo elige su color más barato

# etiquetas permitidas por anillo (baseline)
para cada anillo r:
    si a_r > b_r:      perm[r] = {A}              # mayoría estricta A (forzado)
    elif b_r > a_r:    perm[r] = {B}              # mayoría estricta B (forzado)
    elif a_r == b_r>0: perm[r] = {A, B}           # empate
    else:              perm[r] = {L} ∪ {A si hasA} ∪ {B si hasB}   # sin llaves

# restricción global: cada dueño con llaves necesita >=1 anillo de su color
hay_vacio   = existe algún anillo sin llaves
hay_empate  = existe algún anillo con a_r == b_r > 0
tieneA = existe anillo con A ∈ perm[r] forzado (a_r > b_r)
tieneB = existe anillo con B ∈ perm[r] forzado (b_r > a_r)
penaliza = 0
para cada color C en {A, B} con (hasC y no tieneC):     # dueño con llaves sin anillo
    si hay_vacio o hay_empate:
        penaliza += 0
        fija un anillo vacío/empate a color C  ->  perm[ese anillo] = {C}
    si no:
        r* = argmin_{r con llaves} 2*|a_r - b_r|        # flip más barato
        penaliza += 2*|a_{r*} - b_{r*}|
        perm[r*] = {C}                                  # color fijado ANTES del DP
ops_llave = baseline + penaliza

# --- objetivo 2: maximizar f con DP de árbol ---
bono(x, y) = 2 si x==y∈{A,B} ;  1 si x==y==L ;  0 si no
base(x)    = 1 si x==L ;  0 si no            # cuenta n_L

dp(u, lab):                                  # mejor f del subárbol con u=lab
    val = base(lab)
    para cada hijo c de u:
        val += max_{cl ∈ perm[c]} ( dp(c, cl) + bono(lab, cl) )
    regresa val

f = sum sobre raíces t de  max_{lab ∈ perm[t]} dp(t, lab)
K = (1 si hasA else 0) + (1 si hasB else 0)
ops_anillo = (M + R - K) - f

imprime "Case X:", ops_llave, ops_anillo

Verificación con los ejemplos.

  • Caso 1: \(\sum\min=1\Rightarrow\) ops_llave \(=2\); \(M{+}R{-}K=2{+}3{-}2=3\), \(f_{\max}=2\) (con \(a=B\): arista \(a\)-\(b\) queda \(BB\)) \(\Rightarrow\) 1. Da 2 1.
  • Caso 2: solo Adam, ops_llave \(=0\); \(M{+}R{-}K=0{+}3{-}1=2\), \(f=0\) \(\Rightarrow\) 2. Da 0 2.
  • Caso 3: ambos tienen llaves y \(R=1\) \(\Rightarrow\) impossible.
  • Caso 4: ops_llave \(=0\); \(M{+}R{-}K=4{+}5{-}2=7\), \(f=0\) (no hay aristas monocromáticas) \(\Rightarrow\) 7. Da 0 7.

Complejidad: \(O(R+M+\#\text{llaves})\) por caso (DP lineal sobre el bosque). Memoria \(O(R)\).
