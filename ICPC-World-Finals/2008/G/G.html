ICPC World Finals · 2008

Problema G — Net Loss

PDF original del concurso: ../problems.pdf

El problema

Rose N. Blatt está diseñando una red neuronal embebida para meterla dentro de un celular. Una vez entrenada por el dueño del teléfono, la red le permitirá al usuario dictar mensajes de texto sin usar las manos. La idea central del diseño de Rose es usar funciones de respuesta polinomiales complicadas en cada nodo de la red (en lugar de las funciones de umbral más tradicionales que usan otras redes neuronales).

Cuando Rose estuvo lista para elegir sus polinomios, se topó con un problema: por la poca memoria disponible, no le alcanzaba el espacio para guardar todos los coeficientes de los polinomios de su red. Decidió usar, para cada polinomio, una aproximación en forma de una curva poligonal continua de dos segmentos:

\[ y = a_1 x + a_0 \qquad\text{y}\qquad y = b_1 x + b_0 . \]

Los dos segmentos se juntan en un punto cuya coordenada \(x\), llamada \(c\), está entre \(-1\) y \(+1\). Rose quiere que la aproximación sea la mejor posible en el sentido de que la distancia entre el polinomio \(p\) y la función de aproximación \(g\) sea mínima. Definimos esa distancia como la integral del cuadrado de su diferencia:

\[ d(p,g) = \int_{-1}^{1} \bigl(p(x)-g(x)\bigr)^2 \, dx . \]

Por ejemplo, si el polinomio es \(x^2 - 0.2\), la mejor aproximación poligonal, con las rectas juntándose en \(c = 0\), es la que se muestra en la Figura 2 (la línea punteada es la gráfica de la aproximación poligonal). En ese caso:

\[ g(x) = \begin{cases} -x - 0.367, & -1 \le x \le 0 \\[4pt] \;\;x - 0.367, & 0 \le x \le 1 \end{cases} \]

En los pocos bytes disponibles por nodo, Rose puede guardar los valores de \(a_1\), \(a_0\), \(b_1\), \(b_0\) y \(c\) como números con signo. Por suerte Rose tiene un programa que le da una buena estimación del valor de \(c\). Dado ese valor, te piden ayudarle a encontrar los valores óptimos de \(a_1\), \(a_0\), \(b_1\) y \(b_0\) para las aproximaciones de sus polinomios.

Entrada

El archivo de entrada contiene varios casos de prueba. Cada caso consta de tres líneas:

  • La primera línea trae un entero positivo \(n\), con \(1 \le n \le 10\), que es el grado del polinomio \(p(x)\).
  • La segunda línea trae \(n+1\) números entre \(-1\) y \(1\) inclusive, que son los coeficientes de \(p(x)\) desde el término de mayor orden hasta el término constante, expresados con a lo más tres cifras después del punto decimal.
  • La tercera línea trae el valor de \(c\), con \(-1 < c < 1\), expresado con a lo más tres cifras después del punto decimal.

Después del último caso de prueba viene una línea que contiene el entero cero.

Salida

Para cada caso de prueba, imprime el número de caso (empezando en 1) y los cuatro valores óptimos, mostrando cada uno con exactamente tres cifras después del punto decimal. El primer y el segundo valor son los parámetros \(a_1\) y \(a_0\) del segmento \(y = a_1 x + a_0\) que define a \(g\) en el rango \(-1 \le x \le c\). El tercer y el cuarto valor son los parámetros \(b_1\) y \(b_0\) del segmento \(y = b_1 x + b_0\) que define a \(g\) en el rango \(c \le x \le 1\). La distancia \(d(p,g)\) entre \(p\) y \(g\) (como se definió arriba) debe ser la mínima sobre todas las elecciones posibles de \(a_1\), \(a_0\), \(b_1\) y \(b_0\).

Resumen

Te dan un polinomio \(p(x)\) de grado \(\le 10\) y un punto \(c\in(-1,1)\). Debes hallar una función poligonal continua \(g\) de dos segmentos \(\bigl(a_1x+a_0\text{ en }[-1,c]\) y \(b_1x+b_0\text{ en }[c,1]\bigr)\) que minimice \(\int_{-1}^{1}(p-g)^2\,dx\), y reportar \(a_1,a_0,b_1,b_0\).

Lo que importa de los límites \(n\le 10\) es minúsculo: lo costoso son las integrales, que se resuelven con fórmulas cerradas. La parte central es minimizar una forma cuadrática con una restricción lineal de igualdad ⇒ mínimos cuadrados con un multiplicador de Lagrange (sistema lineal \(5\times 5\)).

Pistas

Pista 1 — ¿Qué tipo de problema es?

El punto de quiebre \(c\) ya te lo dan fijo. ¿Qué pasa entonces con la integral \(\int_{-1}^{1}(p-g)^2\)? ¿Puedes partirla en dos pedazos independientes salvo por una sola condición que los une?

Con \(c\) fijo, cada segmento vive en su propio intervalo. Lo único que conecta a las dos rectas es que deben tocarse en \(x=c\) (continuidad). Sin esa atadura, cada intervalo sería un ajuste por mínimos cuadrados por separado.

Pista 2 — La restricción escondida

¿Cuál es exactamente la condición de continuidad en \(x=c\)? ¿La puedes escribir como una ecuación lineal en \(a_1,a_0,b_1,b_0\)?

La curva debe ser continua: \(a_1 c + a_0 = b_1 c + b_0\). Es una restricción lineal de igualdad sobre las cuatro incógnitas. Sin ella el problema se separaría en dos regresiones triviales; con ella necesitas un método que respete la igualdad.

Pista 3 — Minimizar una cuadrática con una igualdad

El objetivo es una función cuadrática (convexa) de \(a_1,a_0,b_1,b_0\) y la restricción es lineal. ¿Qué herramienta clásica te da directamente el mínimo en ese escenario?

Multiplicadores de Lagrange. Igualar a cero el gradiente del lagrangiano da ecuaciones lineales; junto con la restricción obtienes un sistema lineal pequeño que resuelves exactamente (no hace falta búsqueda ni iteración).

Pista 4 — ¿Qué piezas necesitas calcular?

Para armar las ecuaciones aparecen integrales de \(p(x)\) y de \(x\,p(x)\) sobre \([-1,c]\) y \([c,1]\), más momentos de \(1\) y \(x\). ¿Cómo obtienes \(\int x^k\,dx\) sin error numérico de integración?

Todo es polinomial: \(\int x^k\,dx = x^{k+1}/(k+1)\). Evalúa en los extremos. Así obtienes valores exactos para los momentos \(\int_{-1}^{c} x^j\,dx\), \(\int_{c}^{1} x^j\,dx\) y los términos \(\int x^k p,\ \int x^{k+1} p\).

Pista final — Solución completa (upsolving)

Planteamiento. Queremos minimizar

\[ F = \int_{-1}^{c}\!\!\bigl(p - a_1 x - a_0\bigr)^2 dx \;+\; \int_{c}^{1}\!\!\bigl(p - b_1 x - b_0\bigr)^2 dx \]

sujeto a la continuidad \(h \equiv a_1 c + a_0 - b_1 c - b_0 = 0\).

Momentos. Define sobre el intervalo izquierdo \([-1,c]\): \(M_j = \int_{-1}^{c} x^j\,dx\), o sea \(M_0=c+1,\ M_1=\tfrac{c^2-1}{2},\ M_2=\tfrac{c^3+1}{3}\). Y sobre el derecho \([c,1]\): \(N_j = \int_{c}^{1} x^j\,dx\), o sea \(N_0=1-c,\ N_1=\tfrac{1-c^2}{2},\ N_2=\tfrac{1-c^3}{3}\).

Con \(p(x)=\sum_{k} p_k x^k\), define también

\[ P_0=\!\int_{-1}^{c}\!\! p,\quad P_1=\!\int_{-1}^{c}\!\! x\,p,\quad Q_0=\!\int_{c}^{1}\!\! p,\quad Q_1=\!\int_{c}^{1}\!\! x\,p, \]

cada una sumando términos \(p_k\bigl(\beta^{k+1}-\alpha^{k+1}\bigr)/(k+1)\) (para \(P_0,Q_0\)) o \(p_k\bigl(\beta^{k+2}-\alpha^{k+2}\bigr)/(k+2)\) (para \(P_1,Q_1\)), con \([\alpha,\beta]\) el intervalo correspondiente.

Condiciones de óptimo. Con multiplicador \(\lambda\) y usando \(\mu=\lambda/2\), igualar a cero el gradiente del lagrangiano \(F-\lambda h\) da un sistema lineal de \(5\times 5\) en \((a_1,a_0,b_1,b_0,\mu)\):

\[ \begin{aligned} a_1 M_2 + a_0 M_1 - \mu c &= P_1 \\ a_1 M_1 + a_0 M_0 - \mu &= P_0 \\ b_1 N_2 + b_0 N_1 + \mu c &= Q_1 \\ b_1 N_1 + b_0 N_0 + \mu &= Q_0 \\ a_1 c + a_0 - b_1 c - b_0 &= 0 \end{aligned} \]

Resuélvelo con eliminación gaussiana. Las primeras cuatro ecuaciones son "normales" (mínimos cuadrados) corregidas por el término del multiplicador, y la quinta es la continuidad.

Por qué es correcto. \(F\) es una forma cuadrática convexa (suma de cuadrados), así que su mínimo restringido a un subespacio afín (la recta de continuidad) es único y se caracteriza exactamente por las condiciones KKT de igualdad: \(\nabla F = \lambda\,\nabla h\) más la restricción. Eso es justo el sistema de arriba; cualquier solución del sistema es el mínimo global.

Verificación con el ejemplo. Caso 1: \(p=x^2-0.2,\ c=0\). Sale \(M_0=1, M_1=-\tfrac12, M_2=\tfrac13\), \(P_0=\tfrac13-0.2,\ P_1=-0.15\), y por simetría \(a_0=b_0=-0.367,\ a_1=-1,\ b_1=1\): coincide con Case 1: -1.000 -0.367 1.000 -0.367.

Pseudocódigo 
leer n
mientras n != 0:
    leer coef[0..n]            # coef[0] = termino de mayor grado x^n
    leer c
    # p_k = coeficiente de x^k  (invertir el orden de lectura)
    para k en 0..n: p[k] = coef[n-k]

    # momentos en [-1,c] y [c,1]
    M0 = c+1;        M1 = (c*c-1)/2;   M2 = (c*c*c+1)/3
    N0 = 1-c;        N1 = (1-c*c)/2;   N2 = (1-c*c*c)/3

    # integrales de p y x*p sobre cada intervalo (exactas)
    funcion integralPolinomio(a, b, desplazamiento):   # ∫ x^desplazamiento * p
        s = 0
        para k en 0..n:
            e = k + desplazamiento + 1
            s += p[k]*(b^e - a^e)/e
        regresa s
    P0 = integralPolinomio(-1, c, 0);  P1 = integralPolinomio(-1, c, 1)
    Q0 = integralPolinomio( c, 1, 0);  Q1 = integralPolinomio( c, 1, 1)

    # sistema 5x5: incognitas [a1, a0, b1, b0, mu]
    A = matriz 5x5 de ceros;  rhs = vector 5
    # ec.1: a1*M2 + a0*M1 - mu*c = P1
    A[0]=[M2, M1, 0,  0, -c];  rhs[0]=P1
    # ec.2: a1*M1 + a0*M0 - mu   = P0
    A[1]=[M1, M0, 0,  0, -1];  rhs[1]=P0
    # ec.3: b1*N2 + b0*N1 + mu*c = Q1
    A[2]=[ 0,  0, N2, N1,  c];  rhs[2]=Q1
    # ec.4: b1*N1 + b0*N0 + mu   = Q0
    A[3]=[ 0,  0, N1, N0,  1];  rhs[3]=Q0
    # ec.5: a1*c + a0 - b1*c - b0 = 0  (continuidad)
    A[4]=[ c,  1, -c, -1,  0];  rhs[4]=0

    x = resolverGauss(A, rhs)          # eliminacion con pivoteo
    a1,a0,b1,b0 = x[0],x[1],x[2],x[3]
    imprimir "Case #: a1 a0 b1 b0" con 3 decimales

    leer n

Complejidad: por caso, \(O(n)\) para las integrales y \(O(1)\) para resolver el sistema fijo \(5\times 5\); memoria \(O(n)\). Trivial para \(n\le 10\).
