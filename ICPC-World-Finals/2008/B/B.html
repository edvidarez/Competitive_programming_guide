ICPC World Finals · 2008

Problema B — Always an Integer

PDF original del concurso: ../problems.pdf

El problema

La combinatoria es la rama de las matemáticas que se ocupa de contar objetos discretos. Por ejemplo: ¿de cuántas formas puedes escoger dos personas de un grupo de n? ¿En cuántas regiones se divide un disco circular al conectar entre sí n puntos de su borde? ¿Cuántos cubos hay en una pirámide con capas cuadradas que van de 1×1 hasta n×n?

Muchas preguntas así tienen respuestas que se reducen a polinomios sencillos en n. La respuesta a la primera es n(n-1)/2 = (n²-n)/2. La segunda es (n⁴-6n³+23n²-18n+24)/24. La tercera es n(n+1)(2n+1)/6 = (2n³+3n²+n)/6. Escribimos estos polinomios en una forma estándar: un polinomio con coeficientes enteros dividido entre un denominador entero positivo.

Estos polinomios responden preguntas cuyas respuestas solo pueden ser enteras. Pero como tienen coeficientes fraccionarios, ¡parecería que podrían dar resultados no enteros! Para esos ejemplos en particular, evaluarlos en un entero positivo siempre da un entero. Para otros polinomios de forma parecida, esto no necesariamente se cumple, y puede ser difícil distinguir los dos casos. Esa, naturalmente, es tu tarea.

Entrada

La entrada consta de varios casos de prueba, cada uno en una línea. Cada caso es una expresión de la forma (P)/D, donde P es un polinomio con coeficientes enteros y D es un denominador entero positivo. P es una suma de términos de la forma Cn^E, donde el coeficiente C y el exponente E cumplen:

  1. E es entero con 0 ≤ E ≤ 100. Si E=0, el término se escribe como C. Si E=1, se escribe como Cn, salvo que C sea 1 o -1: en ese caso se escribe n o -n.
  2. C es entero. Si C es 1 o -1 y E no es 0 ni 1, el término aparece como n^E o -n^E.
  3. Solo los C no negativos que no son el primer término llevan un + adelante.
  4. Los exponentes de términos consecutivos son estrictamente decrecientes.
  5. C y D caben en un entero con signo de 32 bits.

La entrada termina con una línea que contiene un solo punto (.).

Salida

Para cada caso, imprime el número de caso (empezando en 1). Luego imprime Always an integer si el polinomio del caso evalúa a un entero para todo entero positivo n; imprime Not always an integer en caso contrario. Un caso por línea, con el mismo formato del ejemplo.

Resumen

Te dan un polinomio P(n) con coeficientes enteros (grado ≤ 100) y un entero positivo D. Decide si P(n)/D es entero para todo entero positivo n, es decir, si D divide a P(n) para todo n. Hay que parsear la expresión y dar el veredicto.

Lo que importa de los límites El grado es ≤ 100, así que el polinomio tiene a lo más 101 coeficientes. C y D caben en 32 bits ⇒ los productos parciales caben en 64 bits si trabajas módulo D. No puedes probar "todos los n"… ¿o sí podrás con muy poquitos?

Pistas (ábrelas una por una)

No abras todas de golpe. Lee una, piénsala, y solo si te atoras abre la siguiente. La última pista es la solución completa.

Pista 1 — Reformula la pregunta

¿Ya pensaste que "P(n)/D es entero" es lo mismo que "D divide a P(n)"? ¿Y que la pregunta es sobre divisibilidad módulo D?

O sea: ¿se cumple P(n) ≡ 0 (mod D) para todo entero positivo n? Trabajar mod D te quita de encima los números gigantes.

Pista 2 — No puedes probar infinitos n… pero quizá pocos basten

¿Cuántos puntos determinan por completo a un polinomio de grado d? ¿Y si esa misma idea sirve para la divisibilidad?

Un polinomio de grado d queda determinado por d+1 valores. Sospecha algo: tal vez baste revisar la divisibilidad en unos pocos n consecutivos para concluir sobre todos.

Pista 3 — La herramienta: diferencias finitas

¿Conoces las diferencias finitas ΔP(n) = P(n+1) − P(n)? ¿Notaste que si P tiene grado d, entonces ΔP tiene grado d−1, y la diferencia (d+1)-ésima es cero?

Además, P(n+1) = P(n) + ΔP(n). Si supieras que D divide a un bloque inicial de valores y a sus diferencias, la divisibilidad se "propaga" hacia adelante (y hacia atrás). ¿A dónde te lleva eso?

Pista 4 — El teorema que resuelve todo

¿Y si te digo que basta con que D divida a P en d+1 enteros consecutivos?

Afirmación: para un polinomio P de grado d con coeficientes enteros, D | P(n) para todo entero n si y solo si D | P(n) para d+1 enteros consecutivos (por ejemplo n = 1, 2, …, d+1). Por la fórmula de Newton, P(n) = Σ Δ^k P(a) · C(n−a, k) con C(·,k) entero; si las d+1 diferencias iniciales son divisibles por D, todo P(n) lo es.

Pista 5 — El otro 50% del problema: el parser

¿Ya tienes claro cómo leer términos como n^E, -n, +11n, C (sin n)?

Recuerda los casos del enunciado: cuando |C|=1 el número no aparece; E=0 se ve como C; E=1 se ve como Cn. Guarda los coeficientes en un arreglo indexado por exponente (tamaño 101). El grado es el mayor E con coeficiente distinto de cero.

Pista final — Solución completa (upsolving)

Idea. P(n)/D es entero para todo n ⟺ D divide a P(n) para todo entero n ⟺ D divide a P(n) en grado+1 enteros consecutivos. Como grado ≤ 100, ¡solo hay que evaluar a lo más 101 valores, todo módulo D!

  1. Parsear (P)/D a un arreglo coef[0..100] (coeficiente por exponente) y leer D. Determinar el grado d.
  2. Para n = 1, 2, …, d+1: calcular P(n) mod D con aritmética modular y verificar que sea 0.
  3. Si todos dieron 0 ⇒ Always an integer; si alguno no ⇒ Not always an integer.
Pseudocódigo 
resolver(linea):
    (coef[0..100], D) = parsear(linea)        # coef por exponente; D > 0
    d = mayor exponente con coef != 0  (si P es 0, es Always)
    para n = 1 .. d+1:
        val = 0
        para e = 0 .. d:
            # término coef[e] * n^e, todo mod D
            val = ( val + (coef[e] mod D) * (n^e mod D) ) mod D
        val = ((val mod D) + D) mod D          # normaliza negativos
        si val != 0:
            regresa "Not always an integer"
    regresa "Always an integer"

# n^e mod D: exponenciación rápida o producto iterativo (e <= 100, baratísimo)

¿Por qué es correcto? Por el teorema de diferencias finitas: si un polinomio de grado d con coeficientes enteros es divisible por D en d+1 enteros consecutivos, entonces lo es en todos los enteros (las d+1 diferencias finitas iniciales quedan divisibles por D, y P(n)=Σ Δ^kP(1)·C(n−1,k) con coeficientes binomiales enteros). Revisar n=1..d+1 cubre, por tanto, a todos los positivos.

Complejidad. O(d²) por caso (≤ ~10⁴ operaciones), trivial.

Cuidados de implementación:

  • Trabaja siempre mod D; nunca evalúes el polinomio completo en enteros (desborda muchísimo).
  • Normaliza los negativos: ((x % D) + D) % D.
  • Productos (coef mod D)·(n^e mod D): con D hasta ~2·10⁹, el producto cabe en 64 bits.
  • El parser es la mitad del problema; cubre ±n^E, Cn, término constante y los signos.
  • Polinomio idéntico a cero ⇒ siempre entero.
